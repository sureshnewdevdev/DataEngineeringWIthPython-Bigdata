2. What is Python?

Python is a simple, readable, high-level programming language. In Data Engineering, Python is used to automate tasks, read files, transform data, call APIs, interact with databases, and write PySpark jobs.

Raw Data
→
Python Logic
→
Clean Data
→
Spark / Analytics

Simple Explanation

Think of Python as a worker who can read data, apply rules, handle mistakes, and prepare output for the next system.

Real-time Data Engineering Example

An e-commerce company receives order files every day. Python can read the order file, validate missing values, calculate order amount, log errors, and pass the clean data to Spark for large-scale processing.

order_quantity = 3
unit_price = 800
total_amount = order_quantity * unit_price
print(f"Total order amount: ₹{total_amount}")

3. Identifiers and Variables

A variable is a named memory location used to store a value. An identifier is the name given to variables, functions, classes, or objects.

Good Variable Names

student_name = "Anu"
order_amount = 2400
is_valid_record = True

Avoid Poor Names

x = "Anu"
a1 = 2400
flag = True

Rules for Identifiers

  • Can contain letters, numbers, and underscore.
  • Cannot start with a number.
  • Cannot use Python keywords like if, for, class.
  • Python is case-sensitive: name and Name are different.

Data Engineering Context

source_file_name = "orders_2026_07_01.csv"
record_count = 5000
failed_records = 12
pipeline_status = "Completed"
Common beginner mistake: Do not use spaces in variable names. Use student_name, not student name.

4. Input and Output Formatting

Formatting helps us display output clearly. In real projects, logs, reports, and console outputs must be readable.

Using input()

name = input("Enter student name: ")
marks = float(input("Enter marks: "))
print(name, marks)

Using f-strings

student_name = "Anu"
marks = 86.5678
print(f"Student: {student_name}")
print(f"Marks: {marks:.2f}")

Output

Student: Anu
Marks: 86.57
Trainer note: Emphasize f-strings. They are clean, modern, and frequently used in logging and report generation.

5. Python Collections

Collections are used to store multiple values. In Data Engineering, we often process multiple records, columns, file names, and configuration values.

CollectionOrdered?Mutable?Allows Duplicate?Use Case
ListYesYesYesStore records, marks, file names
TupleYesNoYesFixed record or coordinates
SetNoYesNoUnique values like course names
DictionaryYesYesKeys are uniqueKey-value data like JSON records

List Example

marks = [72, 85, 91, 66]
marks.append(88)
print(max(marks))
print(sum(marks) / len(marks))

Tuple Example

student_record = (101, "Anu", "Python")
print(student_record[1])

Set Example

courses = {"Python", "SQL", "Python", "Spark"}
print(courses)  # duplicate Python is removed

Dictionary Example

student = {
    "id": 101,
    "name": "Anu",
    "course": "Python",
    "marks": 86.5
}
print(student["name"])
Real-time connection: JSON data looks very similar to a Python dictionary. This is why dictionaries are important before learning APIs and Spark JSON processing.

6. Conditional Statements

Conditional statements help Python make decisions. In data pipelines, conditions are used to validate records, filter rows, and handle business rules.

if / elif / else

marks = 86

if marks >= 90:
    grade = "A"
elif marks >= 75:
    grade = "B"
elif marks >= 50:
    grade = "C"
else:
    grade = "Fail"

print(f"Grade: {grade}")

Nested if

is_file_available = True
record_count = 500

if is_file_available:
    if record_count > 0:
        print("Process the file")
    else:
        print("File is empty")
else:
    print("File missing")
Best practice: Avoid too many nested if blocks. For production code, keep logic readable and split complex conditions into functions.

7. Loops: for, while, range, break, continue

Loops are used to repeat a task. In Data Engineering, we use loops to process files, records, rows, or API pages.

for Loop

marks = [72, 85, 91, 66]
for mark in marks:
    print(mark)

range()

for number in range(1, 6):
    print(number)

while Loop

count = 1
while count <= 5:
    print(count)
    count += 1

Simulated do-while in Python

Python does not have a direct do-while loop, but we can simulate it.

while True:
    user_input = input("Enter yes to continue: ")
    if user_input.lower() != "yes":
        break

break and continue

marks = [45, 66, 82, 95, 70]

for mark in marks:
    if mark < 50:
        continue  # skip failed marks
    if mark > 90:
        print("Top score found", mark)
        break
    print("Passed mark", mark)
Trainer example: Use loops to process many records from a file. Each record can be validated using conditions.

8. Functions, Lambda, map, filter, reduce

A function is a reusable block of code. Functions help us avoid repeating the same logic again and again.

Normal Function

def calculate_grade(mark):
    if mark >= 90:
        return "A"
    elif mark >= 75:
        return "B"
    elif mark >= 50:
        return "C"
    else:
        return "Fail"

print(calculate_grade(86))

Why Functions Matter in Data Engineering

  • Validation function: check whether record is valid.
  • Transformation function: clean customer name or calculate total amount.
  • Utility function: read config, write log, connect to database.

Lambda Function

A lambda function is a small one-line anonymous function.

add_bonus = lambda mark: mark + 5
print(add_bonus(80))

map()

map() applies a function to every item in a collection.

marks = [80, 90, 40]
bonus_marks = list(map(lambda x: x + 5, marks))
print(bonus_marks)

filter()

filter() selects only matching items.

marks = [80, 90, 40]
passed = list(filter(lambda x: x >= 50, marks))
print(passed)

reduce()

reduce() combines items into one result.

from functools import reduce
marks = [80, 90, 40]
total = reduce(lambda x, y: x + y, marks)
print(total)
Bridge to Spark: The idea of applying transformations to data is important. Later, Spark transformations will follow a similar thinking pattern.

9. Exception Handling

Exception handling is used to handle errors gracefully without crashing the program. In data pipelines, exceptions happen because of missing files, invalid data, network failures, or wrong formats.

Try Risky Code
→
Catch Error
→
Log / Recover
→
Continue Safely

try / except / else / finally

try:
    result = 10 / 2
except ZeroDivisionError:
    print("Cannot divide by zero")
else:
    print("Division successful", result)
finally:
    print("This block always runs")

Practical Example

def safe_divide(a, b):
    try:
        return a / b
    except ZeroDivisionError:
        print("Error: denominator cannot be zero")
        return None
    except TypeError:
        print("Error: input should be number")
        return None
    finally:
        print("Division operation completed")
BlockPurpose
tryCode that may fail
exceptCode to handle error
elseRuns only when no exception occurs
finallyAlways runs, usually for cleanup
Production reminder: Never hide errors silently. Handle them, log them, and decide whether to continue or stop the pipeline.

10. Logging

Logging records what happened in a program. In real projects, print statements are not enough. Logs help support teams debug failures and audit pipeline behavior.

LevelMeaningExample
DEBUGDetailed developer informationVariable values, step-by-step internal details
INFONormal successful eventFile processing started or completed
ERRORHandled failureInvalid record or missing column
CRITICALMajor failureDatabase unavailable or source folder missing

Logging Example

import logging

logging.basicConfig(
    filename="app.log",
    level=logging.INFO,
    format="%(asctime)s - %(levelname)s - %(message)s"
)

logging.info("Pipeline started")
logging.error("Invalid record found")
logging.critical("Source file missing")

Real-time Data Engineering Scenario

import logging

file_name = "orders.csv"
record_count = 5000
failed_records = 12

logging.info(f"Started processing file: {file_name}")
logging.info(f"Total records: {record_count}")
logging.error(f"Failed records: {failed_records}")
Trainer explanation: Logging is a professional habit. It makes beginner programs look like production-ready programs.

13. Day 1 Interview Questions and Answers

1. What is Python?

Python is a high-level, interpreted programming language known for simple syntax and readability. It is widely used in automation, data engineering, web development, data science, and scripting.

2. What is the difference between list and tuple?

A list is mutable, meaning we can add, update, and remove values. A tuple is immutable, meaning it cannot be changed after creation.

3. What is the difference between set and dictionary?

A set stores unique values. A dictionary stores key-value pairs where keys are unique.

4. What is the use of if-elif-else?

It is used to execute different blocks of code based on different conditions.

5. What is the use of range()?

range() generates a sequence of numbers. It is commonly used with for loops.

6. What is the difference between break and continue?

break stops the loop completely. continue skips the current iteration and moves to the next one.

7. Why do we use functions?

Functions make code reusable, readable, and easier to maintain.

8. What is lambda?

A lambda is a small anonymous function used for short operations, commonly with map, filter, or sorting logic.

9. Why is exception handling needed?

It prevents the application from crashing and helps handle expected failures gracefully.

10. Why is logging important?

Logging records application events, errors, and status messages. It helps debugging, monitoring, auditing, and production support.

11. What are logging levels?

Common levels are DEBUG, INFO, WARNING, ERROR, and CRITICAL. They classify the seriousness of a message.

12. How does Python help in Data Engineering?

Python helps read files, clean data, transform records, handle errors, connect to databases, call APIs, and write PySpark jobs.
