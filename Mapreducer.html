3. Introduction to MapReduce

MapReduce is a distributed data processing model used in Hadoop.

Simple Definition

MapReduce divides a big data processing job into two major phases: Map phase and Reduce phase.

Technical Definition

MapReduce is a programming model and processing framework that processes large datasets in parallel across a Hadoop cluster. It reads input data from HDFS, processes it using Mapper and Reducer logic, and writes the final output back to HDFS.

Two Main Phases

PhasePurposeSmartRide Example
MapRead input and create intermediate key-value pairsChennai → 1, completed → 1
ReduceGroup same keys and aggregate valuesChennai → 3, completed → 7
Map = Convert raw data into key-value pairs
Reduce = Combine same keys and produce final result

4. Diagram: How MapReduce Works in Hadoop

User / Client | | Submit MapReduce Job v +----------------------+ | YARN ResourceManager | +----------------------+ | | Starts Application Master v +----------------------+ | Application Master | | for MapReduce Job | +----------------------+ | | Requests Containers v +----------------------+ +----------------------+ | Node Manager | | Node Manager | | Container: Mapper | | Container: Mapper | +----------------------+ +----------------------+ | | | Reads HDFS block/split | Reads HDFS block/split v v +----------------------+ +----------------------+ | DataNode | | DataNode | | HDFS Blocks | | HDFS Blocks | +----------------------+ +----------------------+ Mapper Output | v Shuffle and Sort | v +----------------------+ | Container: Reducer | +----------------------+ | | Writes final output v HDFS Output Folder

Very Simple Working

  1. Input file is stored in HDFS.
  2. HDFS stores the file as blocks in DataNodes.
  3. MapReduce creates input splits.
  4. RecordReader reads each split line by line.
  5. Mapper produces intermediate key-value pairs.
  6. Shuffle and Sort groups same keys.
  7. Reducer aggregates values.
  8. Final output is written back to HDFS.

5. Data Flow in MapReduce

Input File: Chennai completed Bangalore completed Chennai completed Step 1: RecordReader (0, "Chennai completed") (18, "Bangalore completed") (38, "Chennai completed") Step 2: Mapper (Chennai, 1) (completed, 1) (Bangalore, 1) (completed, 1) (Chennai, 1) (completed, 1) Step 3: Shuffle and Sort Bangalore -> [1] Chennai -> [1, 1] completed -> [1, 1, 1] Step 4: Reducer Bangalore -> 1 Chennai -> 2 completed -> 3 Step 5: HDFS Output part-r-00000
TermMeaning
InputData stored in HDFS
Mapper OutputIntermediate key-value pairs
ShuffleTransfers mapper output to reducers
SortSorts keys before reducer
Reducer OutputFinal output stored in HDFS
Common Mistake: Mapper output is not final output. Final output is written by Reducer.

6. Mapper Class

The Mapper class reads input records and emits intermediate key-value pairs.

Input KeyInput Value
Byte offsetLine of text
Input:
0, "Chennai completed"

Mapper Output:
Chennai, 1
completed, 1

Java Mapper Code

public static class WordMapper
        extends Mapper<Object, Text, Text, IntWritable> {

    private final static IntWritable one = new IntWritable(1);
    private Text word = new Text();

    protected void map(Object key, Text value, Context context)
            throws IOException, InterruptedException {

        String line = value.toString();
        StringTokenizer tokenizer = new StringTokenizer(line);

        while (tokenizer.hasMoreTokens()) {
            word.set(tokenizer.nextToken());
            context.write(word, one);
        }
    }
}
  • key is the byte offset of the line.
  • value is one line from the input file.
  • context.write(word, one) sends intermediate output.

7. Reducer Class

The Reducer class receives grouped keys and values, then produces final output.

Reducer Input:
Chennai -> [1, 1, 1]
completed -> [1, 1, 1, 1, 1, 1, 1]

Reducer Output:
Chennai -> 3
completed -> 7

Java Reducer Code

public static class WordReducer
        extends Reducer<Text, IntWritable, Text, IntWritable> {

    private IntWritable result = new IntWritable();

    protected void reduce(Text key, Iterable<IntWritable> values, Context context)
            throws IOException, InterruptedException {

        int sum = 0;

        for (IntWritable value : values) {
            sum += value.get();
        }

        result.set(sum);
        context.write(key, result);
    }
}
  • Reducer receives one key at a time.
  • All values of the same key are grouped together.
  • Reducer loops through all values.
  • Reducer writes final key and total count.

8. Driver Program

The Driver Program is the main class that configures and submits the MapReduce job.

  • Create Hadoop configuration
  • Create job object
  • Set Mapper class
  • Set Reducer class
  • Set input path
  • Set output path
  • Set output key and value types
  • Submit job to Hadoop/YARN
public static void main(String[] args) throws Exception {

    Configuration conf = new Configuration();

    Job job = Job.getInstance(conf, "SmartRide WordCount MapReduce");

    job.setJarByClass(SmartRideWordCount.class);
    job.setMapperClass(WordMapper.class);
    job.setReducerClass(WordReducer.class);

    job.setOutputKeyClass(Text.class);
    job.setOutputValueClass(IntWritable.class);

    FileInputFormat.addInputPath(job, new Path(args[0]));
    FileOutputFormat.setOutputPath(job, new Path(args[1]));

    System.exit(job.waitForCompletion(true) ? 0 : 1);
}
Driver Program is like the project setup file. It tells Hadoop what class to run, where input is located, and where output should be stored.

10. Role of RecordReader

RecordReader converts an InputSplit into key-value records for the Mapper.

Mapper cannot directly understand raw file blocks. RecordReader reads the split and gives records in key-value format.

Mapper Input KeyMapper Input Value
Byte offsetOne line of text
Input line:
Chennai completed

RecordReader gives Mapper:
Key   = 0
Value = Chennai completed
InputSplit | v RecordReader | | Creates key-value records v Mapper Example: Raw Data: "Chennai completed" Record: (0, "Chennai completed")
RecordReader acts like a translator between InputSplit and Mapper.

12. Interview Questions and Answers

Q1. What is MapReduce?

MapReduce is a distributed processing model in Hadoop that processes large data using Map and Reduce phases.

Q2. What is Mapper?

Mapper reads input records and produces intermediate key-value pairs.

Q3. What is Reducer?

Reducer receives grouped keys and values and produces final aggregated output.

Q4. What is Driver Program?

Driver Program configures and submits the MapReduce job to Hadoop/YARN.

Q5. What is Shuffle and Sort?

Shuffle transfers mapper output to reducers and Sort orders/group keys before reducer execution.

Q6. What is HDFS Block?

HDFS Block is a physical storage unit used by HDFS to store files.

Q7. What is InputSplit?

InputSplit is a logical processing unit used by MapReduce to assign data to mappers.

Q8. Difference between Block and InputSplit?

Block is for storage; InputSplit is for processing.

Q9. What is RecordReader?

RecordReader converts InputSplit into key-value records for Mapper.

Q10. What are Mapper input key and value in TextInputFormat?

Key is byte offset and value is one line of text.

Q11. Where is final output stored?

Final output is stored in the HDFS output directory.

Q12. Why output directory should not already exist?

MapReduce fails if output path already exists to avoid accidental overwrite.

Q13. Can MapReduce run without Reducer?

Yes, map-only jobs are possible when aggregation is not needed.

Q14. What is Combiner?

Combiner is an optional mini-reducer that runs after mapper to reduce data transfer.

Q15. Why is MapReduce slower than Spark?

MapReduce writes intermediate data to disk, while Spark can process data in memory.

Q16. Which component runs MapReduce job in Hadoop 2.x/3.x?

YARN runs and manages MapReduce jobs using Resource Manager, Node Manager, Application Master and containers.
