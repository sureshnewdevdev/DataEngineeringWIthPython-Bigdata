3. What is RDD?

Simple Definition

RDD stands for Resilient Distributed Dataset. It is the basic data structure in Spark used to store and process data in distributed form.

Technical Definition

An RDD is an immutable, distributed collection of objects partitioned across a cluster. Spark can process these partitions in parallel, and if one partition is lost, Spark can recreate it using lineage information.

LetterMeaningSpoon-feeding Explanation
RResilientFault tolerant. If data is lost, Spark can recompute it from previous steps.
DDistributedData is split into partitions and kept across worker machines.
DDatasetA collection of records such as numbers, strings, rows or objects.

4. RDD Mental Model

Before writing commands, understand how Spark thinks about data.

Immutable
An RDD cannot be modified directly. Every transformation creates a new RDD.
Lazy
Transformations are remembered but not executed immediately.
Partitioned
RDD data is split into partitions for parallel processing.
Fault Tolerant
RDD can be recreated using lineage if any partition is lost.
In-memory Friendly
Repeated RDDs can be cached to improve performance.
Low-level API
RDD gives more control but less automatic optimization than DataFrames.

RDD Lineage Example

Raw RDD
→
filter city
→
map fare
→
count / collect

Spark remembers the relationship between RDDs. This relationship is called lineage. If an executor fails, Spark can use lineage to recompute the missing partition.

Important: In modern Spark projects, DataFrame/Dataset APIs are usually preferred for structured data. But RDD is still important for understanding Spark internals and interview questions.

7. Method 1: Create RDD using Parallelized Collections

Meaning

A parallelized collection means an existing collection from the driver program is distributed into Spark partitions.

Syntax

Scala Spark Shell syntax
val rdd = sc.parallelize(Seq(10, 20, 30, 40), 2)

Here, 2 means we request 2 partitions.

Example 1: Number RDD

spark-shell
val numbers = sc.parallelize(Seq(10, 20, 30, 40, 50), 2)
numbers.collect()
numbers.count()
numbers.getNumPartitions
Expected Output:
Array(10, 20, 30, 40, 50)
5
2

Example 2: SmartRide City RDD

spark-shell
val cities = sc.parallelize(Seq("Chennai", "Bangalore", "Hyderabad", "Chennai"))
val chennaiOnly = cities.filter(city => city == "Chennai")
chennaiOnly.collect()
Memory Point: parallelize is best for learning, testing and small demo datasets. It is not the usual way to load production big data.

PySpark Equivalent

pyspark shell
numbers = sc.parallelize([10, 20, 30, 40, 50], 2)
numbers.collect()
numbers.count()
numbers.getNumPartitions()

8. Method 2: Create RDD using External Datasets

Meaning

An external dataset means data stored outside your program, usually in a file system. Spark can read files from local disk, HDFS, S3-compatible storage and other Hadoop-supported file systems.

Common Syntax

spark-shell
val lines = sc.textFile("/path/to/file.txt")
StorageExample Path
Local filefile:///home/vmuser/data/trips.txt
HDFShdfs:///user/vmuser/trips.txt
Relative pathdata/trips.txt

Read File and Create RDD

spark-shell
val trips = sc.textFile("data/trips.txt")
trips.count()
trips.take(3)

Read HDFS File

HDFS + spark-shell
// First upload the file to HDFS from Linux terminal
// hdfs dfs -mkdir -p /user/vmuser/spark-rdd
// hdfs dfs -put trips.txt /user/vmuser/spark-rdd/

val trips = sc.textFile("/user/vmuser/spark-rdd/trips.txt")
trips.count()
trips.first()
Common Mistake: In Spark, output path should not already exist when using saveAsTextFile. Delete old output path before saving again.

9. Method 3: Create RDD from Existing RDDs

Meaning

Once you already have one RDD, you can create new RDDs by applying transformations. Since RDDs are immutable, Spark does not change the original RDD. It creates a new RDD.

trips RDD
→
filter Chennai
→
chennaiTrips RDD

Example: Filter RDD

spark-shell
val trips = sc.textFile("data/trips.txt")
val chennaiTrips = trips.filter(line => line.contains("Chennai"))
chennaiTrips.collect()

Example: Map RDD

spark-shell
val cities = trips.map(line => line.split(",")(2))
cities.collect()

Example: FlatMap RDD

spark-shell
val words = trips.flatMap(line => line.split(","))
words.take(10)
TransformationPurposeExample Use
mapOne input record becomes one output recordExtract city from each trip line
filterKeep only matching recordsKeep Chennai trips only
flatMapOne input record can become many output recordsSplit lines into words/columns
unionCombine two RDDsCombine weekday and weekend trips
distinctRemove duplicatesGet unique cities

11. Important RDD Operations

1. count()

Returns the number of records in the RDD. It is an action, so it triggers Spark execution.

spark-shell
val numbers = sc.parallelize(Seq(10, 20, 30, 40))
numbers.count()

2. filter()

Creates a new RDD by keeping only records that match a condition. It is a transformation.

spark-shell
val highValues = numbers.filter(x => x > 20)
highValues.collect()

3. take()

Returns the first N records. It is useful for safely previewing data.

spark-shell
numbers.take(2)

4. partitions / getNumPartitions

Partitions are data chunks. Spark creates tasks based on partitions.

spark-shell
numbers.getNumPartitions
numbers.partitions.length

5. cache()

cache() stores an RDD in memory after the first action. Use it when the same RDD is reused multiple times.

spark-shell
val trips = sc.textFile("data/trips.txt")
val chennaiTrips = trips.filter(line => line.contains("Chennai"))
chennaiTrips.cache()
chennaiTrips.count()
chennaiTrips.take(2)

6. saveAsTextFile()

Saves the RDD as text files inside an output directory. Spark writes one output part file per partition.

spark-shell
chennaiTrips.saveAsTextFile("output/chennai-trips")
Output Folder Rule: output/chennai-trips must not already exist. Spark fails if the output directory already exists.

13. Transformations vs Actions

Spark uses lazy evaluation. Transformations create a plan. Actions trigger actual execution.

TypeMeaningRDD ExamplesTriggers Job?
TransformationCreates a new RDDmap, filter, flatMap, distinct, unionNo
ActionReturns result or saves datacount, collect, take, first, saveAsTextFileYes

Lazy Evaluation Example

spark-shell
val trips = sc.textFile("trips.txt")
// No job yet

val chennaiTrips = trips.filter(line => line.contains("Chennai"))
// Still no job yet

chennaiTrips.count()
// Job starts now
Interview Line: Transformations are lazy. Actions are eager because they trigger Spark jobs.

14. Best Practices and Performance Tips

Best PracticeWhy It Matters
Use take() instead of collect() for previewcollect() brings all data to the Driver and can crash it.
Cache only reused RDDsCaching unnecessary RDDs wastes executor memory.
Control partitions wiselyToo few partitions underutilize CPU. Too many partitions increase overhead.
Use DataFrame for structured production dataDataFrames get Catalyst optimization and better SQL support.
Delete output folder before rerunsaveAsTextFile fails if folder already exists.
Avoid parsing errorsCheck split indexes carefully when reading CSV-like text.

16. Interview Questions and Answers

Q1. What is RDD?
RDD stands for Resilient Distributed Dataset. It is Spark's fundamental distributed collection that is immutable, partitioned and fault tolerant.
Q2. Why is RDD called resilient?
RDD is called resilient because Spark can recover lost partitions using lineage information.
Q3. Why is RDD called distributed?
RDD data is split into partitions and stored across workers, allowing parallel processing.
Q4. Is RDD mutable?
No. RDD is immutable. Every transformation creates a new RDD.
Q5. What are the ways to create RDD?
RDD can be created from parallelized collections, external datasets and existing RDDs.
Q6. How do you create RDD from a Scala collection?
Use sc.parallelize(Seq(...)).
Q7. How do you create RDD from a file?
Use sc.textFile("path").
Q8. How do you create RDD from an existing RDD?
Apply transformations such as map, filter, flatMap, union or distinct.
Q9. What is SparkContext?
SparkContext is the entry point for Spark Core and RDD operations. In Spark Shell it is available as sc.
Q10. What is a partition?
A partition is a logical chunk of RDD data. Spark processes partitions in parallel using tasks.
Q11. How do you check number of partitions?
Use rdd.getNumPartitions in Scala Spark Shell or rdd.getNumPartitions() in PySpark.
Q12. What is the difference between transformation and action?
Transformation creates a new RDD lazily. Action triggers execution and returns result or writes output.
Q13. Give examples of transformations.
map, filter, flatMap, distinct, union.
Q14. Give examples of actions.
count, collect, take, first, saveAsTextFile.
Q15. What is lazy evaluation?
Spark does not execute transformations immediately. It waits until an action is called.
Q16. What is lineage?
Lineage is the record of transformations used to build an RDD. Spark uses it for fault recovery.
Q17. What does cache() do?
cache() stores the RDD in memory after the first action so repeated actions can run faster.
Q18. What is saveAsTextFile?
It saves an RDD as text files inside an output directory, generally one part file per partition.
Q19. Why should collect() be avoided on huge data?
Because it brings all RDD records to the Driver and can cause out-of-memory errors.
Q20. RDD vs DataFrame?
RDD is lower-level and gives more control. DataFrame is higher-level, structured and optimized by Spark SQL Catalyst optimizer.
