3. What is RDD?

Simple Definition

RDD stands for Resilient Distributed Dataset. It is a collection of data divided across multiple partitions so that Spark can process those partitions in parallel.

Technical Definition

An RDD is Spark Core's low-level, fault-tolerant distributed collection of elements. It supports parallel operations such as map, filter, reduce, and actions such as count, take, and collect.

Memory Aid: RDD = Recoverable + Divided + Data. Recoverable means Spark can rebuild lost data using lineage. Divided means data is split into partitions. Data means it stores elements such as lines, numbers, objects, or key-value pairs.

RDD in one classroom example

Imagine a trainer gives 1,000 printed trip records to 10 learners. Each learner counts only their bundle. Finally, the trainer combines all counts. In Spark, the trainer is like the Driver, learners are like Executors/Tasks, and bundles are partitions.

4. Important RDD Properties

Resilient
RDD can recover lost partitions using lineage information.
Distributed
Data is split into partitions and processed across worker nodes.
Dataset
It is a collection of records such as strings, numbers, rows, or objects.
Immutable
RDD cannot be changed. Every transformation creates a new RDD.
Lazy
Transformations are planned first and executed only when an action runs.
Partitioned
Parallelism depends strongly on how data is partitioned.

6. Create RDD Using Parallelized Collections

Parallelized collection means you create a normal collection in the Driver program and ask Spark to distribute it as an RDD.

Scala Spark Shell

// Start spark-shell first, then run:
val numbers = sc.parallelize(List(10, 20, 30, 40, 50), 2)

numbers.collect()
numbers.count()
numbers.getNumPartitions
Expected Output: collect() returns all values, count() returns 5, and getNumPartitions returns 2.

PySpark Equivalent

numbers = sc.parallelize([10, 20, 30, 40, 50], 2)

numbers.collect()
numbers.count()
numbers.getNumPartitions()

Trainer Explanation

Driver has data: The list is initially available in the driver process.
Spark distributes: parallelize divides the list into partitions.
Tasks process partitions: When an action is called, tasks process those partitions.
Try it: Change the second parameter from 2 to 4 and check getNumPartitions.

8. Create RDD from Existing RDDs

Most practical RDD work happens by creating one RDD from another RDD. These operations are called transformations. Transformations do not modify the original RDD because RDDs are immutable.

Remove header and filter city

val lines = sc.textFile("file:///home/vmuser/spark-lab/trips.csv")
val header = lines.first()
val data = lines.filter(line => line != header)

val chennaiTrips = data.filter(line => line.contains(",Chennai,"))

chennaiTrips.count()
chennaiTrips.take(10)

Map CSV line to columns

val columnsRDD = data.map(line => line.split(","))

val highFareTrips = columnsRDD.filter(cols => cols(4).toInt > 300)
highFareTrips.map(cols => cols.mkString(" | ")).collect()

Create key-value RDD and aggregate

val cityFareRDD = data.map(line => {
  val cols = line.split(",")
  (cols(2), cols(4).toInt)       // (city, fare)
})

val totalFareByCity = cityFareRDD.reduceByKey((a, b) => a + b)
totalFareByCity.collect()

11. Save RDD Output Using saveAsTextFile

saveAsTextFile writes an RDD as a folder containing one output file per partition, usually named like part-00000, part-00001, etc.

Scala Spark Shell

val highFareText = data
  .map(line => line.split(","))
  .filter(cols => cols(4).toInt > 300)
  .map(cols => cols.mkString(","))

highFareText.saveAsTextFile("file:///home/vmuser/spark-lab/output/high-fare-trips")

Check output in Linux terminal

ls -R /home/vmuser/spark-lab/output/high-fare-trips
cat /home/vmuser/spark-lab/output/high-fare-trips/part-*

Save to HDFS

// Inside spark-shell
highFareText.saveAsTextFile("/spark-lab/output/high-fare-trips")

// Linux terminal
hdfs dfs -ls /spark-lab/output/high-fare-trips
hdfs dfs -cat /spark-lab/output/high-fare-trips/part-*

Before rerunning, delete old output path

// Local path
rm -rf /home/vmuser/spark-lab/output/high-fare-trips

// HDFS path
hdfs dfs -rm -r /spark-lab/output/high-fare-trips
Very Common Error: saveAsTextFile fails if the target output folder already exists. Delete the old folder or use a new output path.

12. Best Practices, Edge Cases and Troubleshooting

Best Practices

  • Use take() for preview. Avoid collect() on large data.
  • Use cache() only when the same RDD is reused multiple times.
  • Understand partitions before tuning performance.
  • Remove CSV headers before numeric conversion.
  • Validate data before using toInt or toDouble.
  • Prefer DataFrames for structured production ETL, but learn RDDs to understand Spark internals.

Common Issues

IssueReasonFix
Input path does not existWrong local/HDFS pathUse ls or hdfs dfs -ls to verify
Output directory already existssaveAsTextFile does not overwriteDelete folder or use a new path
NumberFormatExceptionHeader row or bad numeric dataRemove header and validate columns
Driver out of memoryUsed collect() on huge RDDUse take, saveAsTextFile, or aggregations
Cache seems not workingNo action called after cache()Call count() after cache to materialize
Too many tiny output filesToo many partitionsUse coalesce carefully before saving

Interview Questions and Answers

Q1. What is RDD?
RDD stands for Resilient Distributed Dataset. It is a fault-tolerant distributed collection of elements that can be processed in parallel.
Q2. Why is RDD called resilient?
Because Spark can recover lost RDD partitions using lineage information.
Q3. Why is RDD called distributed?
Because its data is divided into partitions and processed across worker nodes.
Q4. What are the main ways to create RDD?
Using sc.parallelize(), using external datasets like sc.textFile(), and deriving new RDDs from existing RDDs using transformations.
Q5. What is sc.parallelize()?
It converts a local driver collection into a distributed RDD.
Q6. What is sc.textFile()?
It reads a text file and creates an RDD where each element is one line of the file.
Q7. What is transformation?
A transformation creates a new RDD from an existing RDD and is lazily evaluated. Examples: map, filter.
Q8. What is action?
An action triggers Spark execution and returns a result or writes data. Examples: count, take, saveAsTextFile.
Q9. Difference between take and collect?
take(n) brings only n records to Driver. collect() brings all records to Driver, which can be dangerous for large data.
Q10. What does cache() do?
It marks an RDD to be stored in memory after an action executes, so future actions can reuse it.
Q11. Why does saveAsTextFile create a folder?
Because an RDD has multiple partitions, and Spark writes one part file per partition inside the output folder.
Q12. Why does saveAsTextFile fail when folder exists?
Spark does not overwrite output folders by default. Delete the folder or use a new path.
Q13. What is partition in RDD?
A partition is a logical split of RDD data. Each task usually processes one partition.
Q14. How do you check RDD partitions?
In Scala use rdd.getNumPartitions or rdd.partitions.length. In PySpark use rdd.getNumPartitions().
Q15. What is lineage?
Lineage is the chain of transformations used to create an RDD. Spark uses it for fault recovery.
Q16. Is RDD mutable?
No. RDD is immutable. Transformations create new RDDs.
Q17. When should you not use RDD?
For structured data production ETL, DataFrames are usually preferred because Spark can optimize them better.
Q18. What is lazy evaluation?
Spark delays executing transformations until an action is called.
