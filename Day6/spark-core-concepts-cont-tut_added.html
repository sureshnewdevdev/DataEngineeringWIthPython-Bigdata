2. Big Picture: What Happens When a Spark Application Runs?

Assume a ride-hailing company named SmartRide wants to analyze trip data. The trainer writes a Spark program. When the program runs, Spark creates a Driver, requests resources from a cluster manager, starts executors, divides the work into tasks and runs those tasks in parallel.

User Code→Driver Program→SparkContext / SparkSession→Cluster Manager→Executors→Tasks→Result 
// Scala Spark Shell example
val trips = spark.read.option("header", "true").csv("data/trips.csv")
val chennaiTrips = trips.filter($"city" === "Chennai")
chennaiTrips.count()   // action triggers execution
Trainer Explanation: The filter line only builds a plan. The count line is an action. When an action is called, Spark converts the plan into a job, divides the job into stages, and sends tasks to executors.

3. Spark Architecture

Spark applications run as independent processes. The Driver coordinates the application using SparkContext, connects to a cluster manager, gets executors on worker nodes, sends application code to those executors and finally sends tasks to run.

ComponentMeaningSmartRide Example
ApplicationComplete user program built on Spark.Trip revenue analysis application.
Driver ProgramMain control process that creates SparkContext/SparkSession.The process running the analysis code.
Cluster ManagerAllocates resources for Spark applications.YARN gives containers to executors.
Worker NodeMachine where executor processes run.Data node or compute node in the cluster.
ExecutorWorker process that runs tasks and stores cached data.Processes partitions of trips.csv.
TaskSmallest unit of work sent to one executor.Filter partition 1, partition 2, etc.

4. Driver / Executor / Task

Driver

The brain of a Spark application. It maintains the application plan, creates jobs, schedules tasks and collects final results.

Executor

The hands of a Spark application. It executes tasks, stores cached data and sends status/results back to Driver.

Task

The smallest execution unit. Usually one task processes one partition of an RDD/DataFrame.

Real-life Analogy

Spark ComponentHotel Kitchen AnalogyExplanation
DriverHead chefDecides what has to be cooked and assigns work.
Cluster ManagerKitchen managerAllocates kitchen staff and counters.
ExecutorCookActually prepares food.
TaskOne cooking instructionCut vegetables, fry item, make sauce.
PartitionOne tray of ingredientsThe data piece handled by a task.
Important Interview Point: The Driver should not be overloaded. Calling collect() on huge data brings all data back to Driver and can cause Driver out-of-memory errors.

5. Job / Stage / Task Flow

When an action is executed, Spark creates a job. The job is divided into stages. Each stage contains many tasks. Stages are usually separated by shuffle boundaries.

Action→Job→DAG→Stages→Tasks→Executors

Example

val rdd = sc.parallelize(Seq("chennai", "pune", "chennai", "mumbai"), 2)
val result = rdd.map(city => (city, 1))
                .reduceByKey(_ + _)
                .collect()
LineTransformation / ActionExecution?
parallelizeCreates base RDDNo full job yet
mapNarrow transformationLazy
reduceByKeyWide transformation with shuffleLazy, creates stage boundary
collectActionTriggers job
Memory Aid: Action starts JobShuffle splits StagePartition decides Task count

6. SparkContext

Definition: SparkContext is the main entry point for Spark Core. It represents the connection between your application and the Spark cluster. It can create RDDs, accumulators and broadcast variables.

In Spark Shell

// In spark-shell, SparkContext is already available as sc
sc
sc.appName
sc.master
sc.defaultParallelism

Scala Program

import org.apache.spark.{SparkConf, SparkContext}

val conf = new SparkConf()
  .setAppName("SmartRideCoreDemo")
  .setMaster("local[*]")

val sc = new SparkContext(conf)
val data = sc.parallelize(Seq(10, 20, 30, 40))
println(data.count())
sc.stop()

PySpark Program

from pyspark import SparkConf, SparkContext

conf = SparkConf().setAppName("SmartRideCoreDemo").setMaster("local[*]")
sc = SparkContext(conf=conf)
rdd = sc.parallelize([10, 20, 30, 40])
print(rdd.count())
sc.stop()
Important: In one JVM, only one active SparkContext should normally exist. Stop the existing SparkContext before creating another one.

7. SparkConf

Definition: SparkConf is the configuration object used to set application name, master URL and many Spark properties.
val conf = new SparkConf()
  .setAppName("SmartRideTripAnalysis")
  .setMaster("local[4]")
  .set("spark.executor.memory", "2g")
  .set("spark.driver.memory", "1g")

Common Configurations

ConfigurationMeaningExample
spark.app.nameName shown in Spark UI.SmartRideAnalysis

8. SparkSession

Definition: SparkSession is the modern unified entry point for Spark applications. It is used for DataFrames, SQL, Hive integration and can also access SparkContext using spark.sparkContext.

Why SparkSession?

  • It reduces confusion between SparkContext, SQLContext and HiveContext.
  • It is the recommended entry point for modern Spark SQL/DataFrame work.
  • It provides read, sql, catalog and DataFrame APIs.

Scala

import org.apache.spark.sql.SparkSession

val spark = SparkSession.builder()
  .appName("SmartRideSparkSessionDemo")
  .master("local[*]")
  .getOrCreate()

val sc = spark.sparkContext
val df = spark.read.option("header", "true").csv("data/trips.csv")
df.show()

PySpark

from pyspark.sql import SparkSession

spark = SparkSession.builder \
    .appName("SmartRideSparkSessionDemo") \
    .master("local[*]") \
    .getOrCreate()

sc = spark.sparkContext
df = spark.read.option("header", "true").csv("data/trips.csv")
df.show()
ObjectMain UseCurrent Practical Status
SparkContextRDD and low-level Spark Core operationsStill used, available through SparkSession
SparkSessionDataFrames, SQL, Dataset, Hive supportRecommended modern entry point

9. SQLContext and HiveContext

SQLContext and HiveContext are important for interviews because they were used in older Spark versions. In modern Spark, SparkSession has replaced them for most use cases.

SQLContext

Older entry point for DataFrame and SQL operations before SparkSession became the unified entry point.

val sqlContext = new SQLContext(sc)

HiveContext

Older entry point for Hive integration. It extended SQLContext with Hive support.

val hiveContext = new HiveContext(sc)

Modern Hive Support

val spark = SparkSession.builder()
  .appName("SmartRideHiveDemo")
  .master("local[*]")
  .enableHiveSupport()
  .getOrCreate()

spark.sql("SHOW DATABASES").show()
ConceptOlder Spark UsageModern Usage
SQL queriesSQLContextSparkSession
DataFramesSQLContextSparkSession
Hive queriesHiveContextSparkSession + enableHiveSupport()
Memory Aid: SparkSession is the single door. Earlier SQLContext and HiveContext were separate doors.

11. Features of Spark

01

Fast Processing

Spark can keep intermediate data in memory, making iterative and interactive workloads faster than disk-heavy models.

02

Distributed Execution

Data and computation are split across executors on many machines.

03

Fault Tolerance

Spark can recompute lost partitions using lineage information.

04

Lazy Evaluation

Spark waits until an action is called, then optimizes the full execution plan.

05

Multi-language

Supports Scala, Python, Java, SQL and R APIs.

06

Unified Engine

Supports batch, SQL, streaming, machine learning and graph workloads.

Feature Mapping

FeatureBusiness BenefitExample
In-memory cacheFaster repeated analysisCache trip data reused by multiple reports.
Fault toleranceLess manual recoveryExecutor failure can be handled by recomputation.
SQL supportEasy for analystsRevenue by city using SQL.
Streaming supportReal-time decisionsLive fraud/cancellation monitoring.

12. Limitations of Spark

LimitationExplanationHow to Explain in Interview
Needs memorySpark performs well with enough memory. Poor memory sizing causes spills and slow jobs.Fast does not mean unlimited; memory planning is important.
Small file problemToo many tiny files create overhead and too many tasks.Compact files or use optimized storage formats.
Shuffle is expensiveWide transformations move data across network and disk.reduceByKey/groupBy/join need careful tuning.
Driver bottleneckCollecting huge results to Driver can crash it.Avoid collect on huge datasets; write results to storage.
Not required for small dataFor tiny files, Spark startup overhead may be more than processing time.Use pandas/SQL/local processing for small workloads.
Tuning complexityPartitions, memory, executor count and shuffle settings require experience.Spark is powerful but needs monitoring and tuning.
Common Mistake: Saying “Spark has no limitation because it is fast.” Correct answer: Spark is fast for suitable distributed workloads, but memory, shuffle, small files and tuning still matter.

13. Best Practices

1. Prefer SparkSession

Use SparkSession for modern DataFrame, SQL and Hive work.

2. Avoid Huge collect()

Use show(), take(), limit(), or write results to storage.

3. Cache Carefully

Cache only datasets reused multiple times. Unpersist when done.

4. Watch Spark UI

Use Spark UI to understand jobs, stages, tasks, skew and storage.

5. Tune Partitions

Too few partitions underuse cluster. Too many partitions add scheduling overhead.

6. Use Efficient Formats

Use Parquet/ORC for analytics instead of raw CSV where possible.

Cache Example

val cleanedTrips = tripsDF.filter($"fare" > 0).cache()
cleanedTrips.count()
cleanedTrips.groupBy("city").sum("fare").show()
cleanedTrips.unpersist()

15. Interview Questions and Answers

Q1. What is Spark architecture?
Spark architecture consists of a Driver program, SparkContext/SparkSession, cluster manager, worker nodes, executors and tasks. The Driver coordinates the application and executors run tasks on data partitions.
Q2. What is Driver in Spark?
Driver is the main controller of a Spark application. It runs the main program, creates SparkContext, builds DAG, schedules jobs/stages/tasks and communicates with executors.
Q3. What is Executor?
Executor is a worker process launched for a Spark application. It runs tasks, stores cached/persisted data and reports status/results to the Driver.
Q4. What is a Task?
A task is the smallest unit of execution in Spark. Usually one task processes one partition in a stage.
Q5. What is SparkContext?
SparkContext is the connection between a Spark application and the Spark cluster. It is used for Spark Core operations such as creating RDDs, accumulators and broadcast variables.
Q6. What is SparkConf?
SparkConf is used to configure Spark application properties such as application name, master URL, driver memory, executor memory and custom Spark settings.
Q7. What is SparkSession?
SparkSession is the modern unified entry point for Spark. It supports DataFrame, Dataset, SQL and Hive operations, and it gives access to SparkContext.
Q8. SparkContext vs SparkSession?
SparkContext is mainly the low-level Spark Core entry point, especially for RDDs. SparkSession is the modern high-level entry point for SQL/DataFrame/Dataset work and internally provides access to SparkContext.
Q9. What is SQLContext?
SQLContext is the older entry point used for SQL and DataFrame operations before SparkSession became the unified interface.
Q10. What is HiveContext?
HiveContext is the older SQL context with Hive integration. In modern Spark, use SparkSession with enableHiveSupport().
Q11. What is a Job?
A job is a parallel computation triggered by an action such as count(), collect(), save() or show().
Q12. What is a Stage?
A stage is a group of tasks that can run together without shuffle dependency. Wide transformations usually create new stages.
Q13. What is lazy evaluation?
Spark transformations are not executed immediately. Spark waits until an action is called and then creates an optimized execution plan.
Q14. Why should collect() be avoided on huge data?
collect() brings all distributed data to the Driver. For huge datasets, this can cause Driver memory failure.
Q15. What are common limitations of Spark?
Memory requirement, shuffle cost, small file overhead, Driver bottleneck, tuning complexity and unnecessary overhead for small datasets.
Q16. What is Spark UI used for?
Spark UI is used to monitor jobs, stages, tasks, executors, storage, SQL plans and performance issues.
Q17. What is the role of cluster manager?
The cluster manager allocates resources for Spark applications. Examples include Spark Standalone, YARN and Kubernetes.
Q18. Can multiple applications share executors?
Each Spark application generally gets its own executor processes. Data cannot be shared between different SparkContext applications without external storage.
