Trainer Story

Think of Spark as a large classroom. The teacher is the Driver. Groups of students are Executors. The big assignment is split into small parts called partitions. An RDD is the distributed assignment sheet that tells Spark what data exists and how each group should process its part.

Memory Aid: RDD = Resilient + Distributed + Dataset. Resilient means Spark can rebuild lost data using lineage. Distributed means data is split across partitions. Dataset means a collection of records.

2. What is RDD?

RDD stands for Resilient Distributed Dataset. It is Spark Core’s low-level data abstraction: an immutable, partitioned collection of elements that can be processed in parallel across a cluster.

WordMeaningSimple Explanation
ResilientFault tolerantIf a partition is lost, Spark can recompute it from lineage.
DistributedSpread across machinesData is split into partitions and processed in parallel.
DatasetCollection of recordsIt can be numbers, strings, rows, log lines, JSON strings, etc.

Important Properties

Immutable

Once created, an RDD is not changed. Operations create a new RDD.

Lazy

Transformations are not executed immediately. Spark waits until an action is called.

Partitioned

The RDD is divided into partitions. Each partition can be processed in parallel.

Lineage-based

Spark remembers the steps used to create an RDD. This helps recovery.

Interview Point: RDD is low-level and powerful, but for modern application development DataFrames and Datasets are often preferred because they support query optimization. RDD is still very important for understanding Spark internals.

3. RDD Mental Model with Diagram

An RDD is not just data. It is a logical plan containing partitions, dependencies, and lineage information. Spark uses this plan when an action starts execution.

Transformation vs Action

TypeExamplesWhen It RunsReturns

6. Create RDD via Parallelized Collection

Parallelized collection means you already have a local collection in the driver program, and Spark distributes it into partitions.

Scala Spark Shell Example

val numbers = sc.parallelize(Seq(10, 20, 30, 40, 50), 2)
numbers.collect()
numbers.count()
numbers.getNumPartitions

Expected Output

Array[Int] = Array(10, 20, 30, 40, 50)
Long = 5
Int = 2

String RDD Example

val names = sc.parallelize(List("Gopi", "Spark", "Hadoop", "Spark"), 2)
names.count()
names.distinct().collect()

PySpark Equivalent

numbers = sc.parallelize([10, 20, 30, 40, 50], 2)
numbers.collect()
numbers.count()
numbers.getNumPartitions()
Important: Use parallelize mainly for learning, small test data, and unit-like samples. Do not load a very large local list into the driver and then parallelize it; for large data, read from storage using textFile, DataFrame readers, or supported data sources.

8. Create RDD from Existing RDDs

Most real Spark programs create many new RDDs by applying transformations on existing RDDs.

Filter Completed Orders

val lines = sc.textFile("file:///home/vmuser/spark-lab/orders.txt")
val completedOrders = lines.filter(line => line.contains("Completed"))
completedOrders.collect()

Extract Product Names

val products = lines.map(line => line.split(",")(1))
products.collect()

Extract Amounts

val amounts = lines.map(line => line.split(",")(2).toInt)
amounts.collect()
amounts.sum()

flatMap Example: Word Count Preparation

val words = lines.flatMap(line => line.split(","))
words.take(10)
Lineage idea: completedOrders, products, and amounts are new RDDs derived from lines. Spark remembers this relationship.

9. Count and Filter

filter is a transformation. count is an action. This combination is one of the easiest ways to understand Spark lazy execution.

Count Total Records

val lines = sc.textFile("file:///home/vmuser/spark-lab/orders.txt")
lines.count()

Filter Completed Orders and Count

val completed = lines.filter(line => line.contains("Completed"))
completed.count()

Filter Amount Greater Than 20000

val highValueOrders = lines.filter(line => line.split(",")(2).toInt > 20000)
highValueOrders.collect()

Count Products

val productCounts = lines.map(line => line.split(",")(1)).countByValue()
productCounts
CommandTypeMeaning
filterTransformationKeeps only records matching a condition.
countActionReturns number of elements in the RDD.
countByValueActionCounts how many times each value occurs. Use carefully for huge unique values.

10. take / partitions / cache

take

take(n) returns the first n elements to the driver. It is safer than collect() for quick inspection.

lines.take(3)
lines.first()
Warning: Avoid collect() on large RDDs. It brings all data to the driver and can crash the driver memory.

partitions

Partitions decide how work is distributed. More partitions allow more parallelism, but too many tiny partitions increase scheduling overhead.

lines.getNumPartitions
lines.glom().collect()

Change Partitions

val morePartitions = lines.repartition(4)
morePartitions.getNumPartitions

val fewerPartitions = morePartitions.coalesce(2)
fewerPartitions.getNumPartitions

cache

cache() tells Spark to keep an RDD in memory after it is computed, so repeated actions can reuse it.

val completed = lines.filter(line => line.contains("Completed")).cache()
completed.count()   // first action computes and stores
completed.count()   // second action can reuse cached data
CommandUseCare Point
take(n)Preview small number of recordsGood for checking data shape
getNumPartitionsCheck partition countPartition count affects parallelism
cache()Reuse repeatedly accessed RDDUse when reused; otherwise it wastes memory

11. saveAsTextFile

saveAsTextFile is an action that writes an RDD to storage as text. It creates a folder containing part files.

Save Completed Orders

val completed = lines.filter(line => line.contains("Completed"))
completed.saveAsTextFile("file:///home/vmuser/spark-lab/output/completed-orders")

Check Output in Linux Terminal

ls -l /home/vmuser/spark-lab/output/completed-orders
cat /home/vmuser/spark-lab/output/completed-orders/part-*

Save to HDFS

completed.saveAsTextFile("hdfs:///user/vmuser/spark-lab/output/completed-orders")

// Terminal
hdfs dfs -ls /user/vmuser/spark-lab/output/completed-orders
hdfs dfs -cat /user/vmuser/spark-lab/output/completed-orders/part-*
Common Error: If the output folder already exists, saveAsTextFile fails. Delete the folder first or use a new output path.
// Terminal cleanup for local path
rm -rf /home/vmuser/spark-lab/output/completed-orders

// Terminal cleanup for HDFS path
hdfs dfs -rm -r /user/vmuser/spark-lab/output/completed-orders

Single Output File for Demo Only

completed.coalesce(1).saveAsTextFile("file:///home/vmuser/spark-lab/output/completed-single")
Production Warning: coalesce(1) forces output into one partition and can become a bottleneck for large data. Use it only for small demos.

12. Hands-on Lab, Best Practices & Troubleshooting

Hands-on Lab: Order Analysis using RDD

Create input file using the sample orders.txt.
Start Spark Shell using spark-shell.
Create RDD using sc.textFile().
Filter completed orders.
Extract product names and amounts.
Count total records and completed records.
Cache completed orders and run multiple actions.
Save final result using saveAsTextFile().

Complete Spark Shell Script

val lines = sc.textFile("file:///home/vmuser/spark-lab/orders.txt")

println("Total records = " + lines.count())

val completed = lines.filter(line => line.contains("Completed")).cache()
println("Completed records = " + completed.count())

val products = completed.map(line => line.split(",")(1))
println("Completed products:")
products.collect().foreach(println)

val amounts = completed.map(line => line.split(",")(2).toInt)
println("Total completed amount = " + amounts.sum())

completed.saveAsTextFile("file:///home/vmuser/spark-lab/output/final-completed")

Best Practices

Prefer take over collect

Use collect only when you are sure the result is small.

Cache only reused RDDs

Cache helps repeated actions, but unnecessary cache wastes memory.

Use clear paths

Use file:/// or hdfs:/// in labs to avoid path confusion.

Understand partition count

Partitions influence parallelism and output part files.

Common Errors & Fixes

Error / SymptomReasonFix
Input path not foundWrong local/HDFS pathCheck using ls or hdfs dfs -ls.
Output path already existsSpark does not overwrite by defaultDelete output folder or use a new path.
Driver memory errorUsed collect() on huge dataUse take(), saveAsTextFile(), or aggregation.
Slow repeated actionsRDD recomputed each timeUse cache() or persist() when reused.
Too many small output filesToo many partitionsUse suitable partitioning; carefully use coalesce for small output.

13. Interview Questions, Practice Quiz & Timed Exam

Interview Questions and Answers

1. What is RDD?

RDD is Spark’s fault-tolerant, immutable, partitioned collection of elements processed in parallel.

2. Why is RDD resilient?

Because Spark can rebuild lost partitions using lineage information.

3. Is RDD mutable?

No. RDDs are immutable. Transformations create new RDDs.

4. What is lazy evaluation?

Spark delays transformation execution until an action is called.

5. How do you create RDD from collection?

Use sc.parallelize(collection).

6. How do you create RDD from file?

Use sc.textFile(path).

7. What does filter return?

It returns a new RDD containing elements that satisfy the condition.

8. What does count do?

It is an action that returns the number of elements in an RDD.

9. What is partition?

A partition is a logical chunk of an RDD processed by a task.

10. What is cache()?

It stores an RDD after computation so repeated actions can reuse it.

11. Difference between take and collect?

take(n) returns limited records; collect() returns all records to driver.

12. What does saveAsTextFile create?

It creates an output directory with part files, one or more depending on partitions.

13. Why can save fail if path exists?

Spark avoids overwriting output by default for safety.

14. What is lineage?

Lineage is the graph of transformations used to build an RDD.

15. When should RDD be used?

Use RDDs for low-level control, unstructured data, custom transformations, and learning Spark internals.

16. RDD vs DataFrame?

RDD is lower-level; DataFrame is higher-level and optimized by Catalyst.
