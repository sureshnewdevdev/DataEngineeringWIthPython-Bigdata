4. Working with CSV Files

CSV stands for Comma-Separated Values. It is commonly used for reports, exports from Excel, bank statements, HR data, sales data and migration files.

Read CSV Example

import csv

with open("students.csv", "r", newline="", encoding="utf-8") as file:
    reader = csv.DictReader(file)
    for row in reader:
        print(row["student_id"], row["name"], row["course"])

Write CSV Example

import csv

students = [
    {"id": 1, "name": "Arun", "course": "Python"},
    {"id": 2, "name": "Bala", "course": "AI"}
]

with open("output.csv", "w", newline="", encoding="utf-8") as file:
    writer = csv.DictWriter(file, fieldnames=["id", "name", "course"])
    writer.writeheader()
    writer.writerows(students)

5. Working with JSON Files

JSON is widely used for configuration files, API responses, web applications and data exchange between systems.

JSON Example

import json

config = {
    "academy_name": "Tech Learning Academy",
    "currency": "INR",
    "attendance_warning_percent": 75
}

with open("config.json", "w", encoding="utf-8") as file:
    json.dump(config, file, indent=4)

with open("config.json", "r", encoding="utf-8") as file:
    data = json.load(file)
    print(data["academy_name"])
Trainer Tip: Explain JSON as a Python dictionary saved in file format.

7. Working with SQLite Database

SQLite is a lightweight database available with Python. It is useful for learning DB operations without installing SQL Server, MySQL or PostgreSQL.

Create Table and Insert Data

import sqlite3

conn = sqlite3.connect("students.db")
cursor = conn.cursor()

cursor.execute("""
CREATE TABLE IF NOT EXISTS students (
    id INTEGER PRIMARY KEY,
    name TEXT,
    course TEXT,
    fee_paid REAL
)
""")

cursor.execute("INSERT INTO students VALUES (?, ?, ?, ?)", (1, "Arun", "Python", 12000))
conn.commit()
conn.close()

Read Data from DB

import sqlite3

conn = sqlite3.connect("students.db")
cursor = conn.cursor()
cursor.execute("SELECT * FROM students")

for row in cursor.fetchall():
    print(row)

conn.close()

8. Date and Time Modules

Python's datetime module is used for report dates, due dates, time difference, audit logs and scheduling.

Basic Date/Time

from datetime import datetime, date, timedelta

print(date.today())
print(datetime.now())

next_week = date.today() + timedelta(days=7)
print("Next Week:", next_week)

Format Date

from datetime import datetime

now = datetime.now()
print(now.strftime("%d-%m-%Y %I:%M %p"))

Convert String to Date

from datetime import datetime

date_text = "2026-07-03"
converted_date = datetime.strptime(date_text, "%Y-%m-%d")
print(converted_date)

12. Interview Questions and Answers

1. What is the difference between CSV and JSON?

CSV is row and column based, suitable for tabular data. JSON is key-value and nested, suitable for APIs and configuration.

2. Why do we use requirements.txt?

It stores project dependencies so another developer or server can install the same packages using pip install -r requirements.txt.

3. What is the difference between json.load() and json.loads()?

json.load() reads JSON from a file. json.loads() reads JSON from a string.

4. Why use SQLite for learning?

SQLite is built into Python, does not require server installation, and is good for learning SQL and database operations.

5. What is the difference between strftime() and strptime()?

strftime() converts date/time to string. strptime() converts string to date/time.

6. Why should files be opened using with?

The with statement automatically closes the file even if an error occurs.

7. What is newline="" in CSV file handling?

It prevents extra blank lines while reading or writing CSV files, especially on Windows.

8. What is a virtual environment?

A virtual environment isolates project packages from system-level Python packages.

9. What is DictReader?

csv.DictReader reads each CSV row as a dictionary where column names become keys.

10. Give one real-time use case for datetime.

Generating report timestamps, payment due dates, audit logs, event scheduling and SLA calculations.
