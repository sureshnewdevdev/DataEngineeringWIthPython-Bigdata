Module 1

1. Introduction to SQL Window Functions

A SQL Window Function performs a calculation across a set of rows that are related to the current row. The important point is: window functions do not collapse rows.

Simple meaning: GROUP BY gives one row per group, but window functions can show both row-level details and group-level calculations in the same result.

Example Business Question

“Show every order, its order amount, and also show the total sales of that salesperson.” This is a perfect use case for window functions because we need row details and summary values together.

Module 5

5. Aggregate Window Functions

Aggregate window functions calculate totals, averages, counts, minimums, and maximums while keeping each row in the output.

Example 1: Grand Total with Every Row

SELECT
    order_id,
    region,
    salesperson,
    order_amount,
    SUM(order_amount) OVER() AS grand_total
FROM sales_orders;
Explanation: SUM(order_amount) OVER() calculates the total of all orders and repeats that value beside every row.

Example 2: Region-wise Total with Every Order

SELECT
    order_id,
    region,
    salesperson,
    order_amount,
    SUM(order_amount) OVER(PARTITION BY region) AS region_total
FROM sales_orders;
Explanation: PARTITION BY region means the total is calculated separately for South, North, East, and West.

Example 3: Percentage Contribution to Region Sales

SELECT
    order_id,
    region,
    salesperson,
    order_amount,
    ROUND(
      order_amount * 100.0 /
      SUM(order_amount) OVER(PARTITION BY region), 2
    ) AS percentage_of_region_sales
FROM sales_orders;
Real-time use: This is used in dashboards to show how much each order contributes to regional revenue.
Module 6

6. Ranking Window Functions

Ranking functions assign position numbers based on order.

FunctionMeaningTie Handling
ROW_NUMBER()Unique sequence numberNo tie handling; every row gets unique number
RANK()Rank with gapsSame rank for ties, skips next rank
DENSE_RANK()Rank without gapsSame rank for ties, no skipped rank
NTILE(n)Divides rows into n bucketsUseful for quartiles/percentiles

Example: Rank Orders by Amount Within Each Region

SELECT
    order_id,
    region,
    salesperson,
    order_amount,
    ROW_NUMBER() OVER(PARTITION BY region ORDER BY order_amount DESC) AS row_num,
    RANK() OVER(PARTITION BY region ORDER BY order_amount DESC) AS sales_rank,
    DENSE_RANK() OVER(PARTITION BY region ORDER BY order_amount DESC) AS dense_sales_rank
FROM sales_orders;
Trainer Note: Always use ORDER BY with ranking functions. Without order, ranking has no business meaning.
Module 7

7. Top N Records Per Group

This is one of the most important interview and real-time use cases.

Requirement

Find top 2 highest value orders from each region.

WITH ranked_orders AS (
    SELECT
        order_id,
        region,
        salesperson,
        order_amount,
        ROW_NUMBER() OVER(
            PARTITION BY region
            ORDER BY order_amount DESC
        ) AS rn
    FROM sales_orders
)
SELECT *
FROM ranked_orders
WHERE rn <= 2;
Step-by-step:
  1. Partition rows by region.
  2. Sort each region by order amount descending.
  3. Assign row number inside each region.
  4. Filter only row numbers 1 and 2.
Module 8

8. LAG and LEAD Functions

LAG gets data from the previous row. LEAD gets data from the next row. These functions are very useful for trend analysis.

Example: Previous and Next Order Amount by Salesperson

SELECT
    order_id,
    order_date,
    salesperson,
    order_amount,
    LAG(order_amount) OVER(
        PARTITION BY salesperson
        ORDER BY order_date
    ) AS previous_order_amount,
    LEAD(order_amount) OVER(
        PARTITION BY salesperson
        ORDER BY order_date
    ) AS next_order_amount
FROM sales_orders;

Example: Difference From Previous Order

SELECT
    order_id,
    order_date,
    salesperson,
    order_amount,
    order_amount -
    LAG(order_amount) OVER(
        PARTITION BY salesperson
        ORDER BY order_date
    ) AS difference_from_previous_order
FROM sales_orders;
Real-time use: Month-over-month sales comparison, stock price comparison, attendance trend, customer purchase trend.
Module 9

9. FIRST_VALUE and LAST_VALUE

These functions return the first or last value in an ordered window.

SELECT
    order_id,
    salesperson,
    order_date,
    order_amount,
    FIRST_VALUE(order_amount) OVER(
        PARTITION BY salesperson
        ORDER BY order_date
    ) AS first_order_amount,
    LAST_VALUE(order_amount) OVER(
        PARTITION BY salesperson
        ORDER BY order_date
        ROWS BETWEEN UNBOUNDED PRECEDING
             AND UNBOUNDED FOLLOWING
    ) AS last_order_amount
FROM sales_orders;
Important: For LAST_VALUE, use an explicit frame like ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING. Otherwise, many databases return the current row as the last value because of the default frame.
Module 10

10. Window Frames

A window frame controls exactly which rows participate in the calculation for the current row.

Running Total

SELECT
    order_id,
    order_date,
    order_amount,
    SUM(order_amount) OVER(
        ORDER BY order_date
        ROWS BETWEEN UNBOUNDED PRECEDING
             AND CURRENT ROW
    ) AS running_total
FROM sales_orders;

Moving Average of Last 3 Orders

SELECT
    order_id,
    order_date,
    order_amount,
    AVG(order_amount) OVER(
        ORDER BY order_date
        ROWS BETWEEN 2 PRECEDING
             AND CURRENT ROW
    ) AS moving_average_last_3_orders
FROM sales_orders;
FrameMeaning
UNBOUNDED PRECEDINGStart from the first row of the partition
CURRENT ROWCurrent row being calculated
2 PRECEDINGTwo rows before the current row
UNBOUNDED FOLLOWINGGo until the last row of the partition

Module 12

12. Performance and Best Practices

  • Use window functions only when row-level details and analytics are both required.
  • Index columns used in PARTITION BY and ORDER BY when appropriate.
  • Filter unnecessary data before applying window functions.
  • Avoid SELECT * in production queries.
  • Use CTEs to make complex window queries readable.
  • Be careful with LAST_VALUE; define the frame explicitly.
  • Use ROW_NUMBER for deduplication and Top N filtering.

13. Interview Questions and Answers

1. What is a window function?

A window function performs calculations across a set of rows related to the current row without collapsing the output rows.

2. Difference between GROUP BY and window function?

GROUP BY reduces rows into summary rows. Window functions preserve row-level data and add analytical values beside each row.

3. Difference between ROW_NUMBER, RANK, and DENSE_RANK?

  • ROW_NUMBER: unique number for every row.
  • RANK: same rank for ties but skips next rank.
  • DENSE_RANK: same rank for ties and does not skip rank.

4. When do we use LAG?

Use LAG when you need previous row value, such as previous month sales, previous salary, previous order, or previous transaction amount.

5. How do you get Top 2 employees by salary in each department?

WITH ranked_employees AS (
  SELECT
      employee_id,
      department_id,
      salary,
      ROW_NUMBER() OVER(
          PARTITION BY department_id
          ORDER BY salary DESC
      ) AS rn
  FROM employees
)
SELECT *
FROM ranked_employees
WHERE rn <= 2;

6. Why is ORDER BY important in window functions?

ORDER BY defines the sequence of rows. Ranking, running totals, LAG, LEAD, FIRST_VALUE, and LAST_VALUE depend on correct ordering.
