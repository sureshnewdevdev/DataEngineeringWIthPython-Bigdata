2. What is Pandas?

Definition, purpose and real-time use cases.

Pandas is an open-source Python library used for data analysis and data manipulation. It helps you work with tabular and labeled data using two important structures: Series and DataFrame.

Data CleaningData AnalysisCSVExcelSQLReportsETL

Simple Example

Imagine you have an Excel file with employee salary details. With Pandas, you can read the file, remove duplicate records, calculate department-wise average salary and export the final report automatically.

import pandas as pd

data = {
    "employee": ["Anu", "Ravi", "Meena"],
    "department": ["IT", "HR", "IT"],
    "salary": [60000, 45000, 70000]
}

df = pd.DataFrame(data)
print(df)
print(df.groupby("department")["salary"].mean())

Important: Pandas is excellent for structured/tabular data. For very large distributed data, tools like PySpark, Dask or database engines may be more suitable.

4. Series and DataFrame

The two core Pandas data structures.

Series

A Series is a one-dimensional labeled array. Think of it like a single column in Excel with row labels/index.

import pandas as pd

marks = pd.Series([80, 90, 75], index=["Asha", "Ravi", "Meena"])
print(marks)
print(marks["Ravi"])

DataFrame

A DataFrame is a two-dimensional table. It contains multiple columns, and each column is internally a Series.

7. Inspecting, Selecting and Filtering Data

Understand your data before cleaning or analysis.

Sample Dataset

import pandas as pd

sales = pd.DataFrame({
    "order_id": [1, 2, 3, 4, 5],
    "region": ["South", "North", "South", "West", "North"],
    "product": ["Laptop", "Mouse", "Keyboard", "Laptop", "Mouse"],
    "amount": [55000, 700, 1500, 52000, 850]
})

Inspection Methods

print(sales.head())       # first 5 rows
print(sales.tail())       # last 5 rows
print(sales.shape)        # rows and columns count
print(sales.info())       # columns, types, non-null count
print(sales.describe())   # statistical summary

Column Selection

print(sales["amount"])
print(sales[["region", "amount"]])

Filtering Rows

# Orders greater than 1000
high_value = sales[sales["amount"] > 1000]
print(high_value)

# South region orders only
south_orders = sales[sales["region"] == "South"]
print(south_orders)

# Multiple conditions
south_high = sales[(sales["region"] == "South") & (sales["amount"] > 1000)]
print(south_high)

loc vs iloc

MethodMeaningExample

8. Cleaning Data

Handle missing values, duplicates, data types and text problems.

Real-world data is rarely perfect. It may contain missing values, duplicate rows, wrong data types, extra spaces, inconsistent capitalization or invalid numbers.

Sample Dirty Data

import pandas as pd

customers = pd.DataFrame({
    "name": [" Asha ", "RAVI", "Meena", "Asha"],
    "age": [25, None, 30, 25],
    "city": ["chennai", "BENGALURU", None, "chennai"]
})

Find Missing Values

print(customers.isna())
print(customers.isna().sum())

Fill or Remove Missing Values

# Fill missing age with average age
customers["age"] = customers["age"].fillna(customers["age"].mean())

# Fill missing city with Unknown
customers["city"] = customers["city"].fillna("Unknown")

# Remove rows with missing values
# customers = customers.dropna()

Remove Duplicates and Clean Text

customers = customers.drop_duplicates()
customers["name"] = customers["name"].str.strip().str.title()
customers["city"] = customers["city"].str.strip().str.title()
print(customers)

9. Transforming Data

Create new columns, rename columns, apply logic and sort data.

Create New Columns

sales["tax"] = sales["amount"] * 0.18
sales["final_amount"] = sales["amount"] + sales["tax"]
print(sales)

Conditional Column

sales["order_type"] = sales["amount"].apply(lambda x: "High" if x >= 10000 else "Low")
print(sales[["amount", "order_type"]])

Rename Columns

sales = sales.rename(columns={"amount": "order_amount"})

Sort Data

sorted_sales = sales.sort_values(by="order_amount", ascending=False)
print(sorted_sales)

apply() vs Vectorized Operation

ApproachMeaningPerformance
VectorizedColumn-wise operation directly on SeriesUsually faster
apply()Runs custom function row-wise or column-wiseFlexible but can be slower
Best practice: Prefer vectorized operations when possible. Use apply() for custom logic that cannot be expressed directly.

10. GroupBy, Aggregation and Pivot Tables

Summarize data like SQL GROUP BY or Excel PivotTable.

GroupBy Concept

groupby() follows the Split → Apply → Combine pattern.

GroupBy Example

region_sales = sales.groupby("region")["amount"].sum()
print(region_sales)

Multiple Aggregations

summary = sales.groupby("region").agg(
    total_sales=("amount", "sum"),
    average_sales=("amount", "mean"),
    order_count=("order_id", "count")
)
print(summary)

Pivot Table

pivot = pd.pivot_table(
    sales,
    values="amount",
    index="region",
    columns="product",
    aggfunc="sum",
    fill_value=0
)
print(pivot)
Interview point: groupby() is programmatic and flexible; pivot_table() is useful for matrix-like summary reports.

11. Merging, Joining and Concatenating

Combine datasets like SQL joins and file appends.

Merge Example

customers = pd.DataFrame({
    "customer_id": [1, 2, 3],
    "customer_name": ["Asha", "Ravi", "Meena"]
})

orders = pd.DataFrame({
    "order_id": [101, 102, 103],
    "customer_id": [1, 2, 1],
    "amount": [1200, 900, 1500]
})

result = pd.merge(orders, customers, on="customer_id", how="inner")
print(result)

Join Types Diagram

Concatenate Rows

jan = pd.DataFrame({"month": ["Jan", "Jan"], "sales": [1000, 2000]})
feb = pd.DataFrame({"month": ["Feb", "Feb"], "sales": [1500, 2500]})

combined = pd.concat([jan, feb], ignore_index=True)
print(combined)

12. Real-Time Mini Project: Sales Analysis

End-to-end Pandas workflow for business reporting.

Business Scenario

A retail company receives daily sales files from multiple regions. The manager wants a clean monthly report showing region-wise total sales, product-wise sales and high-value orders.

End-to-End Workflow Diagram

Complete Project Code

import pandas as pd

# Step 1: Create sample sales data
sales = pd.DataFrame({
    "order_id": [1, 2, 3, 4, 5, 6],
    "order_date": ["2026-07-01", "2026-07-01", "2026-07-02", "2026-07-02", "2026-07-03", "2026-07-03"],
    "region": ["South", "North", "South", "West", "North", "West"],
    "product": ["Laptop", "Mouse", "Keyboard", "Laptop", "Mouse", "Monitor"],
    "quantity": [1, 3, 2, 1, 5, 2],
    "unit_price": [55000, 700, 1500, 52000, 850, 9000]
})

# Step 2: Convert date column to datetime
sales["order_date"] = pd.to_datetime(sales["order_date"])

# Step 3: Create calculated column
sales["total_amount"] = sales["quantity"] * sales["unit_price"]

# Step 4: Region-wise summary
region_summary = sales.groupby("region").agg(
    total_sales=("total_amount", "sum"),
    total_orders=("order_id", "count"),
    average_order_value=("total_amount", "mean")
).reset_index()

# Step 5: Product-wise summary
product_summary = sales.groupby("product")["total_amount"].sum().reset_index()

# Step 6: High-value orders
high_value_orders = sales[sales["total_amount"] >= 10000]

# Step 7: Export reports
region_summary.to_csv("region_summary.csv", index=False)
product_summary.to_csv("product_summary.csv", index=False)
high_value_orders.to_csv("high_value_orders.csv", index=False)

print("Region Summary")
print(region_summary)
print("\nProduct Summary")
print(product_summary)
print("\nHigh Value Orders")
print(high_value_orders)

13. Best Practices, Interview Q&A, Quiz and Exam

Revision, assessment and interview preparation.

Best Practices

Code Quality

  • Use meaningful DataFrame names.
  • Avoid overwriting raw data too early.
  • Write reusable cleaning functions.
  • Use comments for transformation logic.
  • Use virtual environments for projects.

Performance

  • Prefer vectorized operations.
  • Read only required columns using usecols.
  • Use correct data types to save memory.
  • Process large files in chunks.
  • Use Parquet for analytics workflows when suitable.
