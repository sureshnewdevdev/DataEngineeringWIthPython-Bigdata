1. Introduction & Learning Outcomes

Apache Hive is a data warehouse system built on top of Hadoop-style distributed storage. It allows users to read, manage, and analyze large datasets using an SQL-like language called HiveQL.

Learning Outcomes
By the end of this tutorial, learners should be able to create Hive databases, understand the warehouse directory, create managed and external tables, load data, query data, use partitions, and answer interview questions confidently.

Simple Meaning

Hive is like SQL for big data. Instead of writing complex Java MapReduce programs, we can write SQL-like queries such as SELECT, CREATE TABLE, and GROUP BY.

Where Hive Fits

Storage

Data is usually stored in HDFS or cloud/object storage.

Metadata

Hive stores table and database definitions in the Hive Metastore.

Query

Users write HiveQL; execution may run through engines such as Tez, MapReduce, or Spark depending on setup.

2. Why Hive Database?

A Hive database is a logical namespace used to organize related tables. It is similar to a database or schema in relational database systems, but the actual data is stored as files in distributed storage.

Real-Time Explanation

Imagine a company has sales, customer, product, and clickstream files. Without Hive, each file may be difficult to manage. With Hive, we can create a database such as retail_db and create tables like sales, customers, and products.

NeedHive Helps ByExample
Organizing big dataGrouping tables under databasesCREATE DATABASE retail_db;
SQL-style accessUsing HiveQL instead of low-level codeSELECT category, SUM(amount)
Metadata managementMaintaining schemas in MetastoreTable columns, file location, table type
Batch analyticsProcessing large files efficientlyMonthly sales report
Important: Hive is generally used for analytics and batch processing, not for high-speed OLTP applications such as banking transaction inserts every millisecond.

3. Hive Architecture

Hive architecture separates the SQL interface, metadata, query planning, execution engine, and storage layer.

Key Components

  • Hive CLI / Beeline: Interface where users run HiveQL commands.
  • Driver: Receives queries and manages execution lifecycle.
  • Compiler: Converts HiveQL into an execution plan.
  • Metastore: Stores database, table, column, partition, and location metadata.
  • Execution Engine: Runs the physical job.
  • Storage: Actual data files live in HDFS or compatible storage.

4. Hive Database & Warehouse Concepts

A Hive database is a logical folder-like structure. By default, Hive stores managed table data under the warehouse directory.

Trainer Point: In many Hadoop lab setups, the default warehouse path is /user/hive/warehouse. A database named sales_db may create a directory like /user/hive/warehouse/sales_db.db.

Important Terms

TermMeaningExample
Database / SchemaLogical namespace for tablestraining_db
Warehouse DirectoryDefault location where Hive stores managed table data/user/hive/warehouse
MetastoreRDBMS-backed metadata repositoryStores table schema and location
Managed TableHive manages metadata and data lifecycleCREATE TABLE employee (...)
External TableHive manages metadata; data remains in external locationCREATE EXTERNAL TABLE ... LOCATION ...

Check Warehouse Directory in HDFS

hdfs dfs -ls /user/hive
hdfs dfs -ls /user/hive/warehouse

If the warehouse directory exists, Hive can create database and table directories under it, depending on configuration and table type.

6. Hive Database DDL Commands

DDL stands for Data Definition Language. These commands define database and table structures.

6.1 Show Existing Databases

SHOW DATABASES;

6.2 Create Database

CREATE DATABASE training_db;

6.3 Safe Create with IF NOT EXISTS

CREATE DATABASE IF NOT EXISTS training_db;

6.4 Create Database with Comment and Properties

CREATE DATABASE IF NOT EXISTS retail_db
COMMENT 'Retail analytics database for sales and customer reports'
WITH DBPROPERTIES (
  'created_by'='trainer',
  'purpose'='classroom practice'
);

6.5 Use Database

USE retail_db;

6.6 Describe Database

DESCRIBE DATABASE retail_db;
DESCRIBE DATABASE EXTENDED retail_db;

6.7 Alter Database Properties

ALTER DATABASE retail_db SET DBPROPERTIES (
  'owner'='data_engineering_team',
  'environment'='training'
);

6.8 Drop Database

DROP DATABASE retail_db;

6.9 Drop Database with Tables

DROP DATABASE retail_db CASCADE;
Careful: CASCADE can remove tables inside the database. Use it only in practice environments or when you are fully sure.

Classroom Exercise

CREATE DATABASE IF NOT EXISTS student_db;
SHOW DATABASES;
USE student_db;
DESCRIBE DATABASE EXTENDED student_db;
DROP DATABASE student_db;

7. Managed vs External Tables

Understanding managed and external tables is one of the most important Hive interview topics.

FeatureManaged TableExternal Table
OwnershipHive owns metadata and table data lifecycle.Hive owns metadata only; data is kept in specified external path.
Create SyntaxCREATE TABLECREATE EXTERNAL TABLE
Default LocationWarehouse directoryGiven LOCATION path
Drop BehaviorUsually drops metadata and managed data.Drops metadata; external data is intended to remain.
Best UseHive-controlled temporary or processed datasetsShared data, raw lake data, files used by multiple tools

Managed Table Example

USE retail_db;

CREATE TABLE customers (
  customer_id INT,
  customer_name STRING,
  city STRING
)
ROW FORMAT DELIMITED
FIELDS TERMINATED BY ','
STORED AS TEXTFILE;

External Table Example

CREATE EXTERNAL TABLE ext_customers (
  customer_id INT,
  customer_name STRING,
  city STRING
)
ROW FORMAT DELIMITED
FIELDS TERMINATED BY ','
STORED AS TEXTFILE
LOCATION '/data/retail/customers';

How to Check Table Type

DESCRIBE FORMATTED customers;
DESCRIBE FORMATTED ext_customers;

Look for Table Type in the output. It may show values such as MANAGED_TABLE or EXTERNAL_TABLE.

9. Loading Data & Querying

Sample Local CSV

Create a file named customers.csv:

1,Ravi,Chennai
2,Meena,Bangalore
3,Kumar,Hyderabad
4,Anita,Pune

Create Table

CREATE DATABASE IF NOT EXISTS retail_db;
USE retail_db;

CREATE TABLE customers (
  customer_id INT,
  customer_name STRING,
  city STRING
)
ROW FORMAT DELIMITED
FIELDS TERMINATED BY ','
STORED AS TEXTFILE;

Load Local Data

LOAD DATA LOCAL INPATH '/home/vmuser/customers.csv'
INTO TABLE customers;

Query Data

SELECT * FROM customers;

SELECT city, COUNT(*) AS total_customers
FROM customers
GROUP BY city;

Load Data from HDFS

hdfs dfs -mkdir -p /data/retail
hdfs dfs -put customers.csv /data/retail/

LOAD DATA INPATH '/data/retail/customers.csv'
INTO TABLE customers;
Trainer Tip: LOAD DATA is mainly a file move/copy operation into the table location. It does not transform the file like an ETL mapping tool.

10. Partitions, Bucketing & Performance

Hive performance often depends on how data is physically organized.

Partitioning

Partitioning divides table data into folders based on one or more columns. It is useful when queries frequently filter by that column.

CREATE TABLE sales_partitioned (
  order_id INT,
  customer_id INT,
  amount DECIMAL(10,2)
)
PARTITIONED BY (order_date DATE)
STORED AS ORC;

Insert into Partition

INSERT INTO TABLE sales_partitioned
PARTITION (order_date='2026-07-07')
SELECT order_id, customer_id, amount
FROM sales_raw
WHERE order_date='2026-07-07';

Bucketing

Bucketing distributes data into a fixed number of files based on hash of a column. It can help joins and sampling in selected scenarios.

CREATE TABLE customer_bucketed (
  customer_id INT,
  customer_name STRING,
  city STRING
)
CLUSTERED BY (customer_id) INTO 4 BUCKETS
STORED AS ORC;

Performance Checklist

  • Use columnar formats like ORC or Parquet for analytics.
  • Partition on frequently filtered columns such as date, region, or country.
  • Avoid too many tiny partitions.
  • Run statistics when required for optimizer support.
  • Select only required columns instead of SELECT *.

11. Real-Time Use Case: Retail Analytics Hive Database

This use case is suitable for classroom training and interview explanation.

Business Scenario

A retail company receives daily sales files from multiple stores. The data engineering team wants to store the files in HDFS and analyze sales by city, product, and date using Hive.

Database Design

Complete Practice Script

CREATE DATABASE IF NOT EXISTS retail_db
COMMENT 'Retail analytics classroom database';

USE retail_db;

CREATE TABLE customers (
  customer_id INT,
  customer_name STRING,
  city STRING
)
ROW FORMAT DELIMITED
FIELDS TERMINATED BY ','
STORED AS TEXTFILE;

CREATE TABLE products (
  product_id INT,
  product_name STRING,
  category STRING
)
ROW FORMAT DELIMITED
FIELDS TERMINATED BY ','
STORED AS TEXTFILE;

CREATE TABLE sales (
  order_id INT,
  customer_id INT,
  product_id INT,
  quantity INT,
  amount DECIMAL(10,2)
)
PARTITIONED BY (order_date DATE)
STORED AS ORC;

Business Query

SELECT c.city, p.category, SUM(s.amount) AS total_sales
FROM sales s
JOIN customers c ON s.customer_id = c.customer_id
JOIN products p ON s.product_id = p.product_id
WHERE s.order_date = '2026-07-07'
GROUP BY c.city, p.category;

12. Best Practices & Troubleshooting

Best Practices

  • Use meaningful database names such as retail_db, finance_dw, or training_db.
  • Use IF NOT EXISTS for repeatable classroom/demo scripts.
  • Keep raw data in external tables and curated data in managed/ORC/Parquet tables.
  • Use partitioning for large date-based datasets.
  • Document databases using COMMENT and DBPROPERTIES.
  • Do not run DROP DATABASE ... CASCADE in production unless approved.
  • Use DESCRIBE FORMATTED to inspect table type, location, and storage format.

Common Issues

13. Assessment & Interview Prep

15+ Interview Questions and Answers

1. What is Hive?

Hive is a data warehouse system used to query and manage large datasets stored in distributed storage using HiveQL, an SQL-like language.

2. What is a Hive database?

A Hive database is a namespace that groups related tables. It is also called a schema in HiveQL syntax.

3. What is the default database in Hive?

The default database is named default. If you do not run USE database_name, objects may be created under the default database.

4. What is the Hive warehouse directory?

It is the default storage location where Hive stores managed database and table data, commonly /user/hive/warehouse in lab environments.

5. What is Hive Metastore?

Hive Metastore is the metadata repository that stores information about databases, tables, columns, partitions, table type, and data location.

6. Difference between database and table in Hive?

A database organizes tables. A table defines the schema and storage mapping for actual data files.

7. How do you create a Hive database?

Use CREATE DATABASE database_name; or CREATE DATABASE IF NOT EXISTS database_name;.

8. How do you switch to a database?

Use USE database_name;.

9. How do you list databases?

Use SHOW DATABASES;.

10. How do you see database details?

Use DESCRIBE DATABASE database_name; or DESCRIBE DATABASE EXTENDED database_name;.

11. What is a managed table?

A managed table is a table where Hive manages both metadata and the table data lifecycle.

12. What is an external table?

An external table is a table where Hive manages metadata, but the actual data remains in an external location.

13. When should we use an external table?

Use external tables for shared/raw files or when data is used by multiple tools and should not be deleted when Hive metadata is dropped.

14. What is partitioning?

Partitioning stores table data in separate folders based on partition columns such as date, region, or country to improve query pruning.

15. What is bucketing?

Bucketing splits data into a fixed number of files based on a hash of selected columns, useful in selected join and sampling scenarios.

16. Why is Hive not suitable for OLTP?

Hive is optimized for large-scale analytical queries, not for frequent small row-level transactions like traditional OLTP systems.

17. What command checks table type?

Use DESCRIBE FORMATTED table_name; and inspect the table type field.

18. What happens when you drop a database with CASCADE?

Hive drops the database and the objects inside it. This must be used carefully, especially outside practice environments.
