1. What is Apache Hive?

Apache Hive is a data warehouse tool on top of Hadoop. It allows users to write SQL-like queries called HiveQL to analyze large files stored in HDFS or compatible storage.

Simple Meaning

Hive lets you query big files using SQL instead of writing Java MapReduce programs.

Technical Meaning

Hive compiles HiveQL into execution plans and submits jobs to engines such as MapReduce, Tez, or Spark while using a metastore for table metadata.

Real-time example

A training company stores millions of course enrollment records in HDFS. Instead of coding MapReduce, the data team can run:

SELECT course_name, COUNT(*) AS learner_count
FROM enrollments
GROUP BY course_name;

2. Why Hive needs Hadoop

Hive itself is not the storage layer. Hadoop provides the distributed storage and cluster resource management foundation.

ComponentResponsibilityHive Dependency
HDFSStores files across cluster nodesHive tables usually point to files in HDFS warehouse directories
YARNAllocates cluster resourcesHive execution engines may run jobs on YARN
MapReduce / Tez / SparkProcesses dataHive converts SQL into physical execution work
Metastore DBStores metadataHive stores database, table, schema, partition, and location details
For a beginner local setup, Hive commonly uses an embedded Derby metastore. For production, use a shared RDBMS metastore such as MySQL or PostgreSQL.

3. Prerequisites before starting Hive

Before running Hive commands, confirm the following software and environment variables.

Java

Hadoop and Hive run on JVM. Confirm Java is available.

java -version
echo $JAVA_HOME

Hadoop

Hadoop must be installed and configured.

hadoop version
echo $HADOOP_HOME

Hive

Hive binaries must be accessible.

hive --version
echo $HIVE_HOME

Recommended environment variables

export JAVA_HOME=/usr/lib/jvm/java-8-openjdk-amd64
export HADOOP_HOME=/usr/local/hadoop
export HIVE_HOME=/usr/local/hive
export PATH=$PATH:$HADOOP_HOME/bin:$HADOOP_HOME/sbin:$HIVE_HOME/bin
Trainer note: Paths differ by machine. Always ask learners to run which hadoop, which hive, and which beeline.

Execution flow

  1. User connects using Beeline.
  2. HiveServer2 accepts the SQL query.
  3. Hive checks metadata from Metastore.
  4. Hive creates an execution plan.
  5. The job runs using the configured execution engine.
  6. Data is read from HDFS and results are returned to the client.

5. Step 1 — Start Hadoop HDFS

Hive needs HDFS because Hive tables generally store their data files inside HDFS warehouse locations.

Command

# From any terminal where Hadoop environment is loaded
start-dfs.sh

# Or use full path
$HADOOP_HOME/sbin/start-dfs.sh

Verify using JPS

jps

Expected processes in a single-node setup:

  • NameNode — master metadata service for HDFS
  • DataNode — stores HDFS blocks
  • SecondaryNameNode — checkpoint helper, not a backup NameNode

Verify NameNode UI

http://localhost:9870
Expected result: HDFS should show live DataNodes and safe mode should eventually become OFF.

6. Step 2 — Start YARN

YARN manages resources for processing jobs. Even if a small Hive command appears simple, analytics queries generally need an execution layer.

start-yarn.sh

# Or full path
$HADOOP_HOME/sbin/start-yarn.sh

Verify using JPS

jps

Expected additional processes:

  • ResourceManager — master service for cluster resource allocation
  • NodeManager — worker node agent

Verify ResourceManager UI

http://localhost:8088
Common mistake: Starting Hive before HDFS is ready can cause warehouse path or connection errors.

7. Step 3 — Prepare Hive folders in HDFS

Hive requires temporary and warehouse directories inside HDFS. The warehouse location is where managed table data is stored by default.

hdfs dfs -mkdir -p /tmp
hdfs dfs -mkdir -p /user/hive/warehouse

hdfs dfs -chmod g+w /tmp
hdfs dfs -chmod g+w /user/hive/warehouse

Check folder permissions

8. Step 4 — Initialize Hive Metastore schema

The Hive metastore stores metadata: databases, tables, columns, partitions, storage locations, and file formats. Starting from Hive 2.1, schema initialization is required before using HiveServer2 in a fresh setup.

Beginner local setup using Derby

schematool -dbType derby -initSchema

# Or full path
$HIVE_HOME/bin/schematool -dbType derby -initSchema

Check metastore schema

schematool -dbType derby -info
Important: Derby embedded metastore is suitable for local learning. For multi-user or production setup, use a remote metastore database such as MySQL or PostgreSQL.

9. Step 5 — Start Hive CLI or HiveServer2

There are two common ways beginners start Hive. For modern practice, use HiveServer2 + Beeline.

OptionCommandWhen to useRecommendation

Option A — Start old Hive CLI

hive

# Inside Hive shell
SHOW DATABASES;

Option B — Start HiveServer2

Open Terminal 1:

hiveserver2

# Or full path
$HIVE_HOME/bin/hiveserver2

Keep this terminal running. HiveServer2 is a server process.

Expected result: HiveServer2 should start and listen on port 10000 by default.

10. Step 6 — Connect using Beeline

Open Terminal 2 and connect to HiveServer2.

beeline -u jdbc:hive2://localhost:10000

If your environment asks for username, use:

beeline -u jdbc:hive2://localhost:10000/default -n $USER

Embedded Beeline mode for quick testing

beeline -u jdbc:hive2://

11. Hands-on HiveQL lab after startup

After Hive starts, run this mini lab to confirm that the system can create a database, create a table, load data, and run a group query.

1. Create sample CSV file on local machine

cat > /tmp/employees.csv <<'EOF'
1,Gopi,Trainer,80000
2,Ravi,Developer,65000
3,Meena,Developer,70000
4,Anu,Tester,55000
5,Kumar,Trainer,75000
EOF

2. Create database and table in Hive

CREATE DATABASE IF NOT EXISTS trainingdb;
USE trainingdb;

CREATE TABLE IF NOT EXISTS employees (
  id INT,
  name STRING,
  role STRING,
  salary DOUBLE
)
ROW FORMAT DELIMITED
FIELDS TERMINATED BY ','
STORED AS TEXTFILE;

3. Load local file into Hive table

LOAD DATA LOCAL INPATH '/tmp/employees.csv'
INTO TABLE employees;

4. Query data

SELECT * FROM employees;

SELECT role, COUNT(*) AS total_people, AVG(salary) AS avg_salary
FROM employees
GROUP BY role;

12. Common troubleshooting

Most Hive startup issues happen because Hadoop is not running, HDFS folders are missing, metastore is not initialized, or HiveServer2 is not listening.

Error / SymptomReasonFix
Connection refused jdbc:hive2://localhost:10000HiveServer2 is not running or wrong portRun hiveserver2 in Terminal 1 and retry Beeline
Version information not found in metastoreMetastore schema not initializedRun schematool -dbType derby -initSchema
Permission denied: /user/hive/warehouseHive user does not have HDFS write permissionCreate warehouse folder and apply proper permissions
Name node is in safe modeHDFS is still recoveringWait for safe mode to exit, or in lab use hdfs dfsadmin -safemode leave
start-dfs.sh: command not foundHadoop sbin is not in PATHRun $HADOOP_HOME/sbin/start-dfs.sh or update PATH
FAILED: SemanticException table not foundWrong database selectedRun SHOW DATABASES;, USE database_name;, then SHOW TABLES;

13. Practice, Exam, Interview Q&A and Notes

A. Startup Checklist

B. 15+ Interview Questions and Answers

1. What is Hive?

Hive is a data warehouse system on Hadoop that lets users query large datasets using HiveQL, a SQL-like language.

2. Why do we start HDFS before Hive?

Hive table data is usually stored in HDFS. If HDFS is down, Hive may fail to access warehouse folders and table files.

3. What is Hive Metastore?

Hive Metastore stores metadata such as databases, tables, columns, partitions, file formats, and HDFS locations.

4. What is HiveServer2?

HiveServer2 is a server service that allows clients such as Beeline, JDBC, and ODBC tools to execute Hive queries.

5. What is Beeline?

Beeline is a JDBC command-line client used to connect to HiveServer2 and run HiveQL statements.

6. What is the default HiveServer2 port?

The default HiveServer2 TCP port is commonly 10000.

7. What command initializes Hive metastore schema?

schematool -dbType derby -initSchema for local Derby setup.

8. Why is Hive CLI not preferred now?

Hive CLI is legacy/deprecated compared to HiveServer2 and Beeline, which support multi-client and better security integration.

9. What is the default Hive warehouse path?

A common default warehouse path is /user/hive/warehouse in HDFS.

10. What is the difference between managed and external tables?

For managed tables, Hive manages table data lifecycle. For external tables, Hive manages metadata while data remains at an external location.

11. What happens when we run LOAD DATA LOCAL?

Hive copies data from the local filesystem of the HiveServer2 machine into the target Hive table location.

12. What is YARN's role in Hive?

YARN provides resource management for distributed execution engines used by Hive.

13. What is Derby in Hive setup?

Derby is a lightweight embedded database often used as a local Hive metastore for learning or single-user demos.

14. Why should production not use embedded Derby?

Embedded Derby is not suitable for multi-user concurrent access. Production environments need a shared metastore DB.

15. Which command verifies Hadoop daemons?

jps shows running Java processes such as NameNode, DataNode, ResourceManager, NodeManager, and HiveServer2.

16. What is a Hive partition?

A partition divides table data into subdirectories based on column values, improving query filtering and scan efficiency.

17. What does SHOW DATABASES do?

It lists available Hive databases/schemas registered in the metastore.
