1. Overview

A Hive database is a logical namespace stored in the Hive Metastore. It groups tables, views, functions, properties, and default storage locations. It is not an OLTP database server in the same sense as MySQL or SQL Server; Hive is designed mainly for large-scale analytical workloads.

Metadata

Database and table definitions live in the metastore.

Data

Managed or external files live in HDFS or another supported object/file store.

Execution

Hive compiles HiveQL and runs work through an execution engine.

Real-world case: the HR analytics team needs a company_hr database and an employees table for employee reporting, salary corrections, and inactive-record clean-up.

3. Prerequisites and Connection

HDFS, the Hive Metastore, and HiveServer2 should be running. Use Beeline rather than the legacy Hive CLI for a server-based setup.

Ubuntu service checks and Beeline connection
jps
ss -ltnp | grep -E '9083|10000'

# Start services only when they are not already running.
nohup hive --service metastore > ~/metastore.log 2>&1 &
nohup hive --service hiveserver2 > ~/hiveserver2.log 2>&1 &

beeline -u 'jdbc:hive2://localhost:10000/default' -n vmuser
Do not start duplicates: confirm ports 9083 and 10000 before launching services. A second process can fail or create confusing logs.

ACID session settings

Run before creating or using the transactional table
SET hive.support.concurrency=true;
SET hive.txn.manager=org.apache.hadoop.hive.ql.lockmgr.DbTxnManager;
SET hive.exec.dynamic.partition.mode=nonstrict;
SET hive.enforce.bucketing=true; -- older releases only

In a production cluster, administrators normally place the transaction and compactor settings in configuration files. Session settings cannot replace missing metastore compactor configuration.

4. Create a Hive Database

Simple create
CREATE DATABASE company_hr;
Safe and descriptive create
CREATE DATABASE IF NOT EXISTS company_hr
COMMENT 'Training database for employee CRUD operations'
WITH DBPROPERTIES (
  'created_by'='ItTechGenie Training',
  'purpose'='Hive CRUD demonstration'
);

IF NOT EXISTS

Makes the command repeatable by preventing an error when the database already exists.

DBPROPERTIES

Stores descriptive governance metadata such as owner team, environment, and purpose.

Location note: omit LOCATION for the first lab so Hive uses the configured warehouse path. Add a custom HDFS location only after permissions and governance rules are understood.

6. Update Database Metadata with ALTER

Add or change properties
ALTER DATABASE company_hr SET DBPROPERTIES (
  'environment'='training',
  'owner_team'='data_engineering'
);

DESCRIBE DATABASE EXTENDED company_hr;
Other supported metadata changes
ALTER DATABASE company_hr SET OWNER USER vmuser;

-- Supported in suitable Hive releases:
ALTER DATABASE company_hr
SET LOCATION 'hdfs:///user/hive/custom/company_hr.db';
Important: changing a database location does not move existing tables or partitions. It changes the default parent location used by future objects.

7. Delete a Database

RESTRICT — safer default

Fails when objects remain
USE default;
DROP DATABASE company_hr RESTRICT;

CASCADE — destructive

Drops contained objects
USE default;
DROP DATABASE company_hr CASCADE;
Safety sequence: run USE company_hr; SHOW TABLES;, verify the database name, switch to default, and only then consider CASCADE.

8. Create the Employee Transactional Table

Row-level UPDATE and DELETE require an ACID-capable table. This sample uses ORC, the transactional property, and bucketing for broad compatibility with older Hive distributions.

Create table
USE company_hr;

CREATE TABLE employees (
  employee_id   INT,
  employee_name STRING,
  department    STRING,
  salary        DECIMAL(12,2),
  joining_date  DATE,
  employment_status STRING
)
CLUSTERED BY (employee_id) INTO 4 BUCKETS
STORED AS ORC
TBLPROPERTIES ('transactional'='true');
Verify table metadata
SHOW TABLES;
DESCRIBE employees;
DESCRIBE FORMATTED employees;
SHOW TBLPROPERTIES employees;

10. Read Rows with SELECT

Read all rows
SELECT *
FROM employees
ORDER BY employee_id;
Filter and sort
SELECT employee_id, employee_name, salary
FROM employees
WHERE department = 'Data Engineering'
ORDER BY salary DESC;
Department summary
SELECT department,
       COUNT(*) AS employee_count,
       ROUND(AVG(salary), 2) AS average_salary
FROM employees
GROUP BY department
ORDER BY department;
Production habit: preview an UPDATE or DELETE condition by running it first as a SELECT with the same WHERE clause.

11. Update Existing Rows

Increase Data Engineering salaries by 10%
-- Preview first.
SELECT employee_id, employee_name, salary
FROM employees
WHERE department = 'Data Engineering';

UPDATE employees
SET salary = salary * 1.10
WHERE department = 'Data Engineering';
Activate one employee
UPDATE employees
SET employment_status = 'ACTIVE'
WHERE employee_id = 105;
Restrictions: Hive UPDATE cannot directly change partition columns or bucketing columns. Subqueries in the assigned expression are also unsupported.

13. How Hive ACID Works

HDFS does not update bytes in the middle of an existing file. Hive therefore records mutations as new transactional delta data and merges the base plus relevant deltas when reading.

base_0001
original ORC rows
+
delta_0002
insert/update changes
+
delete_delta_0003
deleted row IDs
→
Reader
current snapshot
Many delta files
→
Compactor
→
New consolidated base
→
Faster future reads
Inspect or request compaction
SHOW TRANSACTIONS;
SHOW COMPACTIONS;

-- Normally automatic; run manually only when appropriate.
ALTER TABLE employees COMPACT 'minor';
ALTER TABLE employees COMPACT 'major';
Auto-commit: Hive's language operations are committed individually. Traditional multi-statement BEGIN/COMMIT/ROLLBACK workflows are not generally available in the same way as OLTP databases.

14. Optional CSV Bulk Load Through a Staging Table

Use a plain text stage for landing data, then insert into the transactional ORC target. This separates file parsing from optimized storage.

Create the CSV staging table
CREATE TABLE employees_stage (
  employee_id INT,
  employee_name STRING,
  department STRING,
  salary DECIMAL(12,2),
  joining_date DATE,
  employment_status STRING
)
ROW FORMAT DELIMITED
FIELDS TERMINATED BY ','
STORED AS TEXTFILE
TBLPROPERTIES ('skip.header.line.count'='1');
Load and convert
LOAD DATA LOCAL INPATH '/home/vmuser/Desktop/employees-sample.csv'
OVERWRITE INTO TABLE employees_stage;

SELECT * FROM employees_stage;

INSERT INTO TABLE employees
SELECT * FROM employees_stage;
HiveServer2 path rule: with LOCAL, the file must be accessible on the HiveServer2 machine, not merely on a remote browser or client computer.

17. Best Practices and Edge Cases

Before mutation

  • Run the WHERE clause as SELECT.
  • Check row counts.
  • Confirm the active database.
  • Use qualified names in automation.

After mutation

  • Verify changed rows.
  • Monitor compactions.
  • Refresh statistics after major changes.
  • Audit who changed data.

Use Hive for analytics

Batch corrections and warehouse maintenance are appropriate. Thousands of tiny per-second updates are not.

Avoid accidental overwrite

Prefer INSERT INTO unless replacement is explicitly required.

Protect destructive DDL

Use RESTRICT by default and grant CASCADE privileges carefully.

Edge case: once a table is marked transactional, do not expect to turn the property back to false. Plan transactional storage intentionally.

Table is not transactionalMissing transactional=true or incompatible tableCreate a managed ORC transactional target and load data into it.
Cannot drop non empty databaseRESTRICT behaviorDrop/move contained objects or deliberately use CASCADE after inspection.
Permission denied on warehouse pathHDFS ownership or modeCheck hdfs dfs -ls, database location, owner, and permissions.
LOCAL file not foundPath exists on client but not HiveServer2 hostCopy the file to the server or use an HDFS path without LOCAL.
Update of bucket/partition column rejectedPhysical layout column cannot be updatedInsert corrected rows into a rebuilt table or partition instead.
