# 7. DOUBLE-CLICK-WORD NOTES  [MUST] - IMPLEMENT EXACTLY AS DESCRIBED            #
################################################################################
This is the most-often-broken feature. Follow this behavior precisely:

TRIGGER
- A note is created ONLY when the user DOUBLE-CLICKS a WORD inside tutorial body
  text. On dblclick the browser selects the word; read it with
  window.getSelection().toString().trim().
- If the selection is empty or contains whitespace (i.e. not a single word), do
  nothing. Do NOT trigger on single click or on empty space.
- Do NOT trigger inside interactive elements or UI: buttons, links, inputs,
  textareas, labels, quiz/exam options, checkboxes, the notes panel, the TOC.
  (Guard with element.closest(...) checks.)

WHAT HAPPENS (NOT a textarea dump)
- Wrap ONLY that one selected word in a <span class="noted-word"> using the
  selection Range (range.surroundContents). Give it a distinct highlight style
  (e.g. yellow marker underline) clearly different from normal text and from hover.
- Open a small POPUP anchored near that word with a text field to type the note,
  plus Save / Delete / Cancel. (Cancel with no text removes the empty highlight.)
  [MUST] If the popup is position:fixed, position it with VIEWPORT coordinates
  from getBoundingClientRect (do NOT add window.scrollY / scrollX, or the popup
  will open far off-screen once the page is scrolled). Clamp it inside the visible
  viewport so it is always seen. Verify this by imagining the user scrolled far
  down before double-clicking.
- Store notes as an ARRAY of objects in localStorage, each:
  { id, word, note, sectionId, sectionTitle, timestamp }.
- NEVER paste the element's whole innerText into a textarea. A note is tied to ONE
  word only.

NOTES PANEL (structured, collapsible)  [MUST show it]
- The panel is always visible (e.g. in a sidebar). List each note as its own row
  showing: the word, the note text, the section it came from, and a timestamp,
  each with its own Edit and Delete buttons, plus a "Clear all" button. This is a
  LIST of rows - NOT a single textarea.
- [MUST] JUMP TO WORD: clicking a note row navigates to the associated highlighted
  word in the tutorial - scroll it into view (accounting for the sticky-header
  offset so it is not hidden underneath the header) and briefly flash/emphasise
  that word so the user can see exactly which one it is. If the word's highlight
  is missing for any reason, re-apply it first, then scroll.

REQUIRED BEHAVIOUR SUMMARY (all four MUST work):
  1. Double-click a WORD -> a popup lets the user add a note.
  2. A word that has a note is visibly HIGHLIGHTED.
  3. The notes LIST is shown with every saved note.
  4. Clicking a note in the list NAVIGATES to (scrolls to + flashes) that word.

HIGHLIGHT + RESTORE
- A word that has a note stays highlighted. Hover/click the highlighted word to
  view/edit/delete its note. Deleting a note removes that word's highlight.
- On page load, RE-APPLY highlights to all saved words. Do this reliably: for each
  saved note, locate the correct occurrence of that word within its section
  (store an occurrence index at creation time and re-find the Nth occurrence on
  reload) and wrap it again.
