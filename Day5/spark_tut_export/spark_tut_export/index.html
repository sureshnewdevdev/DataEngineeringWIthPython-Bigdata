3. What is Apache Spark?

Simple Definition

Apache Spark is a fast distributed data processing engine used to process large data using multiple machines.

Technical Definition

Apache Spark is an open-source distributed computing framework used for batch processing, SQL analytics, real-time streaming, machine learning, graph processing and ETL workloads.

Spoon-feeding Explanation: If one laptop cannot process huge trip data, Spark divides the data into partitions and gives the work to multiple executors. Each executor processes a small part of the data. Finally Spark combines the result.

Where Spark is Used

ScenarioExample
Batch ProcessingAnalyze yesterday's completed trips
StreamingAnalyze live driver locations from Kafka
SQL AnalyticsRevenue by city using SQL query
Machine LearningPredict trip fare or cancellation chance
Graph ProcessingAnalyze customer-driver-city relationships

4. Hadoop, MapReduce and Spark

Hadoop

Hadoop is a big data ecosystem. It mainly contains HDFS for storage, YARN for resource management and MapReduce for processing.

MapReduce

MapReduce is an older Hadoop processing model. It works using map phase and reduce phase. It is reliable but slower because it writes intermediate data to disk.

Spark

Spark is a modern processing engine. It is faster because it can keep intermediate data in memory.

PointHadoop MapReduceSpark
Processing StyleDisk basedIn-memory when possible
SpeedSlower for iterative jobsFaster for iterative and interactive workloads
CodeMore boilerplateLess code
StreamingNot naturalStreaming supported
SQLUsually Hive is usedSpark SQL available
Machine LearningSeparate tools requiredMLlib available
Memory Point: Hadoop is an ecosystem. MapReduce is an old processing engine. Spark is a faster processing engine.

Driver

The Driver is the main controller of a Spark application. It creates SparkSession or SparkContext, converts user code into jobs, creates DAG, divides jobs into stages and schedules tasks.

Executor

Executors are worker processes. They run tasks, store cached data and return results to the Driver.

Task

A Task is the smallest unit of execution. Usually, one task processes one partition.

8. SparkContext, SparkConf and SparkSession

SparkContext

SparkContext is the older entry point for Spark Core. It connects the application to the Spark cluster and is commonly used for RDD operations.

rdd = sc.parallelize([10, 20, 30, 40])
print(rdd.map(lambda x: x * 2).collect())

SparkConf

SparkConf stores Spark application configuration like app name, master URL, driver memory and executor memory.

from pyspark import SparkConf, SparkContext

conf = SparkConf() \
    .setAppName("SmartRideAnalysis") \
    .setMaster("local[*]")

sc = SparkContext(conf=conf)

SparkSession

SparkSession is the modern unified entry point introduced in Spark 2.x. It is recommended for DataFrame, SQL and Hive operations.

from pyspark.sql import SparkSession

spark = SparkSession.builder \
    .appName("SmartRideTripAnalysis") \
    .master("local[*]") \
    .getOrCreate()

SparkContextRDD and Spark Core entry pointStill available
SparkConfConfiguration objectStill used
SparkSessionUnified entry pointRecommended

9. SQLContext and HiveContext

SQLContext

SQLContext was used in older Spark versions for SQL and DataFrame operations.

from pyspark.sql import SQLContext
sqlContext = SQLContext(sc)

HiveContext

HiveContext was used in older Spark versions to connect Spark with Hive metastore and run Hive queries.

Modern Approach

Today, SparkSession replaces SQLContext and HiveContext for most practical use cases.

spark = SparkSession.builder \
    .appName("HiveExample") \
    .enableHiveSupport() \
    .getOrCreate()
Old ComponentUsed ForModern Replacement
SQLContextSQL and DataFrameSparkSession
HiveContextHive integrationSparkSession + enableHiveSupport()

PySpark Program

from pyspark.sql import SparkSession
from pyspark.sql.functions import sum, avg, count

spark = SparkSession.builder \
    .appName("SmartRideSparkIntro") \
    .master("local[*]") \
    .getOrCreate()

trips_df = spark.read.csv("data/trips.csv", header=True, inferSchema=True)

trips_df.show()

trips_df.groupBy("city").agg(sum("fare").alias("total_fare")).show()
trips_df.groupBy("city").agg(count("*").alias("trip_count")).show()
trips_df.groupBy("driver_id").agg(avg("rating").alias("avg_rating")).show()

spark.stop()

Step-by-Step Execution

Create SparkSession: Starts the Spark application.
Read CSV: Loads SmartRide trip data into a DataFrame.
Group by City: Calculates total fare and trip count.
Group by Driver: Calculates average driver rating.
Stop Spark: Releases Spark resources.
Expected Learning Outcome: Learners understand how SparkSession reads data, how transformations are written and how actions trigger execution.

11. Limitations and Best Practices

Limitations

LimitationExplanation
Needs memorySpark is fast because of memory, but low memory can reduce performance.
Small file problemToo many small files create too many tasks and overhead.
Not needed for tiny dataSmall data can be processed with normal Python or SQL.
Requires tuningExecutor memory, driver memory, partitions and shuffle need tuning.
Debugging is harderErrors can happen in Driver, Executor, input data or cluster configuration.

Best Practices

  • Use Spark only when data is large enough.
  • Prefer DataFrame and Spark SQL for structured data.
  • Avoid collecting huge data to the Driver.
  • Cache only reused datasets.
  • Use proper file formats like Parquet or ORC in real projects.
  • Monitor Spark UI to understand jobs, stages and tasks.
Important: Do not use collect() on huge data. It brings all data to the Driver and may crash the application.

12. Interview Questions and Answers

Q1. What is Apache Spark?
Apache Spark is a distributed data processing engine used for batch, streaming, SQL analytics, machine learning and graph processing.
Q2. Why is Spark faster than MapReduce?
Spark is faster because it performs in-memory processing, while MapReduce writes intermediate data to disk.
Q3. What is Spark Core?
Spark Core is the base engine that handles task scheduling, memory management, fault tolerance and RDD operations.
Q4. What is Spark SQL?
Spark SQL is used to process structured data using SQL queries and DataFrames.
Q5. What is Spark Streaming?
Spark Streaming is used to process real-time data from sources such as Kafka, socket and files.
Q6. What is MLlib?
MLlib is Spark's machine learning library used for classification, regression, clustering and recommendation.
Q7. What is GraphX?
GraphX is Spark's graph processing library for relationship-based data.
Q8. What is SparkR?
SparkR allows R language users to work with Spark for big data analytics.
Q9. What is Driver?
Driver is the main controller of a Spark application. It creates jobs, stages and tasks.
Q10. What is Executor?
Executor is a worker process that runs tasks and stores cached data.
Q11. What is a Task?
A task is the smallest unit of execution in Spark.
Q12. What is SparkContext?
SparkContext is the older entry point to Spark Core and is used to create RDDs.
Q13. What is SparkSession?
SparkSession is the modern unified entry point for Spark DataFrame, SQL and RDD operations.
Q14. What is SQLContext?
SQLContext is the older entry point for SQL and DataFrame operations.
Q15. What is HiveContext?
HiveContext is the older component used to connect Spark with Hive.
