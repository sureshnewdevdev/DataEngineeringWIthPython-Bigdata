2. Evolution of Big Data

Big Data evolved because organizations started generating more data than traditional systems could store, process, and analyze efficiently.

Simple Story

Initially, companies stored data in files and databases. Later, websites, mobile apps, sensors, social media, digital payments, and clickstreams started producing huge volumes of data every second. The data was too large, too fast, and too varied for only traditional relational systems.

3. What is Hadoop?

Hadoop is an open-source framework used to store and process very large datasets across a cluster of low-cost commodity machines.

Simple definition: Hadoop allows big files to be split into blocks, stored across many machines, and processed in parallel.

Technical Definition

Apache Hadoop is a distributed storage and batch processing framework. In Hadoop 1.x, its two core components are HDFS for distributed storage and MapReduce for distributed batch processing.

HDFS

Hadoop Distributed File System. Stores large files by splitting them into blocks and distributing those blocks across DataNodes.

MapReduce

Processing model that divides work into map tasks and reduce tasks. It processes data close to where the data is stored.

Real-time Example

An e-commerce company may generate terabytes of clickstream logs daily. Hadoop can store those logs in HDFS and use MapReduce jobs to calculate popular products, user behavior, and failed checkout patterns.

# Example HDFS commands hdfs dfs -mkdir /retail hdfs dfs -put orders.csv /retail/orders.csv hdfs dfs -ls /retail hdfs dfs -cat /retail/orders.csv | head

4. Why Hadoop Was Needed

Traditional databases are excellent for structured transactions, but they were not designed to cheaply store and process petabytes of mixed-format data across hundreds or thousands of machines.

Key Problems Hadoop Solved

  • Storage problem: How to store huge files across many machines?
  • Processing problem: How to process big data without moving all data to one server?
  • Fault tolerance problem: How to continue working when a machine fails?
  • Cost problem: How to use commodity hardware instead of only expensive servers?

6. Characteristics of Big Data

Big Data is commonly explained using the Vs. The most common are Volume, Velocity, Variety, Veracity, and Value.

Volume

Massive amount of data, such as TBs or PBs of logs, transactions, or videos.

Velocity

Speed at which data is produced and must be processed, such as real-time payments.

Variety

Different formats like tables, JSON, images, videos, sensor data, and logs.

Veracity

Trustworthiness and quality of data. Dirty or duplicate data reduces confidence.

Value

Business benefit produced from data, such as fraud detection or customer insights.

Extra Vs

Some discussions also include Variability, Visualization, and Validity.

8. Hadoop 1.x Architecture

Hadoop 1.x architecture has two major parts: HDFS for storage and MapReduce v1 for processing. It follows master-slave architecture.

Main Components

NameNode

Master of HDFS. Stores metadata such as file names, permissions, block IDs, and block locations. It does not store actual file data.

DataNode

Slave node of HDFS. Stores actual data blocks and sends heartbeat and block reports to the NameNode.

Secondary NameNode

Helps with checkpointing by merging fsimage and edit logs. It is not a hot standby for NameNode.

JobTracker

Master of MapReduce v1. Accepts jobs, splits them into tasks, schedules tasks on TaskTrackers, and monitors progress.

TaskTracker

Slave process that runs map and reduce tasks on worker nodes and reports status to JobTracker.

Client

Submits data read/write requests to HDFS and submits MapReduce jobs to JobTracker.

9. HDFS Architecture

HDFS stores a large file by dividing it into blocks. Each block is replicated across different DataNodes to provide fault tolerance.

Large File
→

→

→

→
Fault Tolerance

How Write Works in HDFS

  1. Client asks NameNode for permission to write a file.
  2. NameNode checks metadata and returns target DataNodes.
  3. Client splits the file into blocks.
  4. Blocks are written to DataNodes using a replication pipeline.
  5. DataNodes send acknowledgements.
  6. NameNode updates metadata.

Important HDFS Terms

  • Block: A large file is divided into fixed-size chunks called blocks.
  • Replication factor: Number of copies of each block. Common default in classic Hadoop setups is 3.
  • Heartbeat: DataNode sends regular signals to NameNode to say it is alive.
  • Block report: DataNode reports which blocks it stores.
  • Rack awareness: Hadoop tries to place replicas across racks to reduce failure risk.

10. MapReduce Architecture

MapReduce is a batch processing model. It divides a large job into smaller tasks and runs them across multiple machines.

MapReduce Word Count Example

Suppose we want to count words from millions of log lines.

Input hello hadoop hello big data Mapper Output (hello, 1), (hadoop, 1), (hello, 1), (big, 1), (data, 1) Shuffle and Sort hello -> [1, 1] hadoop -> [1] big -> [1] data -> [1] Reducer Output hello -> 2 hadoop -> 1 big -> 1 data -> 1

Hadoop 1.x MapReduce Components

  • JobTracker: One master process. Receives job request, creates tasks, schedules tasks, and monitors them.
  • TaskTracker: Runs on worker nodes. Executes map/reduce task slots.
  • Data locality: Hadoop tries to run processing near the data blocks to reduce network transfer.
Limitation of Hadoop 1.x: JobTracker handled both resource management and job scheduling. This became a scalability bottleneck, which led to YARN in Hadoop 2.x.

11. Batch vs Streaming Data

Batch processing handles data in groups at scheduled intervals. Streaming processing handles data continuously as it arrives.

PointBatch DataStreaming Data
Processing styleData collected first, processed later.Data processed continuously as it arrives.
LatencyMinutes, hours, or days.Milliseconds to seconds.
ExampleMonthly billing, daily sales report.Live fraud detection, stock prices, cab tracking.
Classic Hadoop fitVery suitable because Hadoop MapReduce is batch-oriented.Not suitable in Hadoop 1.x alone; needs tools like Kafka, Spark Streaming, or Flink.
CostUsually simpler and cheaper.More complex because systems must run continuously.

12. Interview Q&A + Best Practices

Best Practices for Learning Hadoop

  • Always explain Hadoop as two core ideas first: distributed storage and distributed processing.
  • Clearly say Hadoop 1.x uses HDFS and MapReduce v1.
  • Do not say Secondary NameNode is a backup NameNode. It is mainly used for checkpointing.
  • Use examples: log analysis, clickstream analysis, telecom call data, and retail sales analysis.
  • For architecture questions, draw master and slave nodes before explaining.

15+ Interview Questions and Answers

1. What is Hadoop?

Hadoop is an open-source distributed framework used to store and process huge datasets across multiple machines using HDFS and MapReduce.

2. What are the two main components of Hadoop 1.x?

HDFS for distributed storage and MapReduce v1 for distributed batch processing.

3. What is HDFS?

HDFS stands for Hadoop Distributed File System. It stores large files by splitting them into blocks and distributing them across DataNodes.

4. What is NameNode?

NameNode is the master of HDFS. It stores metadata such as file names, directories, block IDs, and block locations.

5. Does NameNode store actual data?

No. NameNode stores metadata. Actual file blocks are stored in DataNodes.

6. What is DataNode?

DataNode is a worker node in HDFS that stores actual data blocks and sends heartbeat signals to NameNode.

7. What is Secondary NameNode?

Secondary NameNode performs checkpointing by merging fsimage and edit logs. It is not a real-time backup NameNode.

8. What is block in HDFS?

A block is a fixed-size chunk of a large file stored in HDFS.

9. Why does HDFS use replication?

Replication provides fault tolerance. If one DataNode fails, another replica can be used.

10. What is MapReduce?

MapReduce is a distributed batch processing model with two main phases: map and reduce.

11. What is JobTracker?

JobTracker is the master process in Hadoop 1.x MapReduce. It accepts jobs, schedules tasks, and monitors execution.

12. What is TaskTracker?

TaskTracker is the worker process that runs map and reduce tasks assigned by JobTracker.

13. What is data locality?

Data locality means moving computation close to the data instead of moving large data over the network.

14. What is the difference between batch and streaming?

Batch processes collected data at intervals. Streaming processes data continuously as it arrives.

15. Is Hadoop 1.x suitable for real-time streaming?

No. Hadoop 1.x MapReduce is mainly batch-oriented. Streaming requires tools like Kafka, Spark Streaming, or Flink.

16. What is the limitation of Hadoop 1.x?

JobTracker is a single point of bottleneck for resource management and job scheduling. Hadoop 2.x solved this using YARN.

17. What are the types of data in Big Data?

Structured, semi-structured, and unstructured data.

18. What are the 5Vs of Big Data?

Volume, Velocity, Variety, Veracity, and Value.
