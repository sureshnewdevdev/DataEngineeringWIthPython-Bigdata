1. Introduction

String functions are used to clean, standardize, combine, split, validate, and extract text values in data pipelines. In real data engineering projects, customer names, emails, product codes, order IDs, and campaign tags often arrive in messy string formats.

Raw text data → PySpark string functions → Clean curated columns → Analytics-ready output

4. concat and concat_ws

df.select(
    "CustomerID",
    concat(col("CustomerName"), lit(" - "), col("Email")).alias("CustomerContact")
).show(truncate=False)

df.select(
    "CustomerID",
    concat_ws("-", col("CustomerID"), col("OrderCode")).alias("CustomerOrderKey")
).show(truncate=False)
Best practice: Use concat_ws when a separator is required. It makes business keys more readable.

5. split and explode

df.select(
    "CustomerID",
    split(col("Items"), ",").alias("ItemArray")
).show(truncate=False)

df.select(
    "CustomerID",
    "OrderCode",
    explode(split(col("Items"), ",")).alias("SingleItem")
).show(truncate=False)
"mobile,charger,cable" → split() → ["mobile","charger","cable"] → explode() → 3 rows

6. startswith and endswith

df.select(
    "OrderCode",
    col("OrderCode").startswith("ORD").alias("IsOrder"),
    col("OrderCode").startswith("RET").alias("IsReturn")
).show(truncate=False)

df.select(
    "Email",
    col("Email").endswith(".com").alias("IsDotComEmail")
).show(truncate=False)

7. initcap, upper, lower

df.select(
    "CustomerName",
    initcap(trim(col("CustomerName"))).alias("ProperName"),
    upper(trim(col("CustomerName"))).alias("UpperName"),
    lower(col("Email")).alias("LowerEmail")
).show(truncate=False)

8. substring and length

df.select(
    "OrderCode",
    substring(col("OrderCode"), 5, 4).alias("OrderYear"),
    "StoreCode",
    substring(col("StoreCode"), 4, 3).alias("CityCode"),
    length(col("StoreCode")).alias("StoreCodeLength")
).show(truncate=False)
Important: PySpark substring starts from position 1, not 0.

9. lpad, ltrim, rtrim, trim

df.select(
    "CustomerName",
    ltrim(col("CustomerName")).alias("LeftTrimmed"),
    rtrim(col("CustomerName")).alias("RightTrimmed"),
    trim(col("CustomerName")).alias("FullyTrimmed")
).show(truncate=False)

df.select(
    "CustomerID",
    lpad(col("CustomerID").cast("string"), 5, "0").alias("PaddedCustomerID")
).show()

10. regexp_extract and regexp_replace

df.select(
    "Email",
    regexp_extract(col("Email"), r"@(.+)$", 1).alias("EmailDomain"),
    "OrderCode",
    regexp_extract(col("OrderCode"), r"([0-9]{4})", 1).alias("ExtractedYear")
).show(truncate=False)

df.select(
    "OrderCode",
    regexp_replace(col("OrderCode"), "-", "").alias("OrderCodeNoDash"),
    regexp_replace(trim(col("CustomerName")), r"\s+", " ").alias("CleanName")
).show(truncate=False)

11. Real-World Retail Use Case

Requirement

Clean retail order data by fixing customer names, normalizing emails, extracting order year and city code, identifying order/return type, and converting comma-separated item values into one item per row.

Expected Result

OutputDescription
CleanCustomerNameTrimmed and title-cased customer name
EmailDomainDomain extracted from email
TransactionTypeORDER, RETURN, or UNKNOWN
OrderYearYear from order code
CityCodeCity extracted from store code
SingleItemOne item per row
result_df = (
    df.withColumn("CleanCustomerName", initcap(trim(col("CustomerName"))))
      .withColumn("EmailLower", lower(col("Email")))
      .withColumn("EmailDomain", regexp_extract(col("EmailLower"), r"@(.+)$", 1))
      .withColumn("TransactionType",
          when(col("OrderCode").startswith("ORD"), "ORDER")
          .when(col("OrderCode").startswith("RET"), "RETURN")
          .otherwise("UNKNOWN"))
      .withColumn("OrderYear", substring(col("OrderCode"), 5, 4))
      .withColumn("CityCode", substring(col("StoreCode"), 4, 3))
      .withColumn("PaddedCustomerID", lpad(col("CustomerID").cast("string"), 5, "0"))
      .withColumn("SingleItem", explode(split(col("Items"), ",")))
)
result_df.show(truncate=False)

12. Interview Q&A

#QuestionAnswer
1concat vs concat_ws?concat joins directly; concat_ws joins with separator.
2Use of split?Converts a string into an array.
3Use of explode?Converts array elements into multiple rows.
4Why use lower for email?For consistent comparison and grouping.
5What does initcap do?Capitalizes first letter of each word.
6What does substring do?Extracts a part of a string.
7PySpark substring starts from?Position 1.
8What is length used for?String length validation.
9What does lpad do?Adds padding on the left side.
10trim vs ltrim vs rtrim?trim removes both sides; ltrim left; rtrim right.
11regexp_extract use?Extract pattern from text.
12regexp_replace use?Replace pattern in text.
13Why avoid Python UDFs?Built-in Spark functions are optimized and distributed.
14Real use of startswith?Classify order IDs such as ORD and RET.
15Real use of explode?Create one row per purchased item.
