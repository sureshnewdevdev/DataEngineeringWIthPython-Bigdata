2. What is RDD?

Core Concept

An RDD is a distributed collection of records. Spark splits a large dataset into partitions, sends those partitions to worker nodes, and processes them in parallel.

Large Dataset
→
Partition 1
Partition 2
Partition 3
→
Parallel Processing
RDD FeatureMeaningSimple Example
ResilientCan recover lost data using lineage.If a partition fails, Spark recalculates it.
DistributedData is split across machines.1 TB file split into many partitions.
DatasetCollection of records.Lines from a file, words, transactions.
ImmutableOriginal RDD is not changed.filter() creates a new RDD.
LazyExecution waits until an action.map() waits until collect().
Interview line: RDD is Spark’s low-level immutable distributed dataset abstraction. It supports fault tolerance through lineage and parallel processing through partitions.

4. Creating RDDs in Python

Hands-on

You can create RDDs mainly in three ways: from Python collections, from files, or from DataFrames.

4.1 Create RDD from Python List

numbers = [10, 20, 30, 40, 50]
rdd = sc.parallelize(numbers)
print(rdd.collect())

4.2 Create RDD from Text File

lines_rdd = sc.textFile("data/input.txt")
print(lines_rdd.take(5))

4.3 Convert DataFrame to RDD

df = spark.createDataFrame([(1, "Gopi"), (2, "Anu")], ["id", "name"])
rdd_from_df = df.rdd
print(rdd_from_df.collect())
MethodUse Case
sc.parallelize()Learning, testing, small in-memory collections.
sc.textFile()Read text/log/CSV-like files as lines.
df.rddConvert DataFrame rows to RDD for low-level processing.

5. RDD Transformations

Most Important

A transformation creates a new RDD from an existing RDD. Transformations are lazy. They do not execute immediately.

RDD
→
map/filter/flatMap
→
New RDD
→
No execution yet

5.1 map()

Applies a function to each element.

rdd = sc.parallelize([1, 2, 3, 4])
squared = rdd.map(lambda x: x * x)
print(squared.collect())   # [1, 4, 9, 16]

5.2 filter()

rdd = sc.parallelize([10, 15, 20, 25, 30])
even_numbers = rdd.filter(lambda x: x % 2 == 0)
print(even_numbers.collect())   # [10, 20, 30]

5.3 flatMap()

flatMap() is useful when one input can produce multiple outputs.

lines = sc.parallelize(["hello hadoop", "hello big data"])
words = lines.flatMap(lambda line: line.split(" "))
print(words.collect())
# ['hello', 'hadoop', 'hello', 'big', 'data']

5.4 distinct(), union(), intersection()

a = sc.parallelize([1, 2, 2, 3])
b = sc.parallelize([3, 4, 5])
print(a.distinct().collect())
print(a.union(b).collect())
print(a.intersection(b).collect())
TransformationPurposeExample Output
mapOne input → one output1 → 1, 2 → 4
flatMapOne input → many outputs"hello spark" → hello, spark
filterKeep matching recordsOnly even numbers
distinctRemove duplicates[1,2,2] → [1,2]

6. RDD Actions

Execution Trigger

An action triggers actual execution and returns result to the driver or writes output to storage.

Transformations
→
DAG Created
→
Action Called
→
Execution Starts
rdd = sc.parallelize([10, 20, 30, 40])
print(rdd.count())        # 4
print(rdd.first())        # 10
print(rdd.take(2))        # [10, 20]
print(rdd.collect())      # [10, 20, 30, 40]
print(rdd.reduce(lambda a, b: a + b))  # 100
Important: Avoid collect() on huge datasets because it brings all data to the driver machine. Use take(), count(), or save output instead.
ActionPurpose
collect()Return all records to driver. Use only for small data.
count()Count records.
take(n)Return first n records.
first()Return first record.
reduce()Aggregate records using a function.
saveAsTextFile()Write output to folder.

7. Pair RDDs: Key-Value Processing

Aggregation

A Pair RDD stores data as key-value pairs like (customer_id, amount) or (word, 1). Pair RDDs are used heavily in grouping, counting, joining, and aggregation.

7.1 reduceByKey()

sales = sc.parallelize([
    ("Chennai", 1000),
    ("Bangalore", 1500),
    ("Chennai", 2000),
    ("Mumbai", 1200)
])

city_total = sales.reduceByKey(lambda a, b: a + b)
print(city_total.collect())
# [('Chennai', 3000), ('Bangalore', 1500), ('Mumbai', 1200)]

7.2 groupByKey()

grouped = sales.groupByKey().mapValues(list)
print(grouped.collect())
Best practice: Prefer reduceByKey() for aggregation because it combines data locally before shuffle. groupByKey() can move more data across network.

7.3 sortByKey()

marks = sc.parallelize([("Ravi", 80), ("Anu", 95), ("Bala", 70)])
print(marks.sortByKey().collect())

7.4 join()

customers = sc.parallelize([(1, "Gopi"), (2, "Priya")])
loans = sc.parallelize([(1, 500000), (2, 300000)])
joined = customers.join(loans)
print(joined.collect())
# [(1, ('Gopi', 500000)), (2, ('Priya', 300000))]

8. Mini Project: Word Count Using RDD

Classic Interview

Word count is the most common RDD interview example because it explains flatMap, map, shuffle, and reduceByKey.

Input: hello hadoop hello big data flatMap: hello, hadoop, hello, big, data map: (hello,1), (hadoop,1), (hello,1), (big,1), (data,1) shuffle + reduceByKey: hello -> [1,1] -> 2 hadoop -> [1] -> 1 big -> [1] -> 1 data -> [1] -> 1
text = sc.parallelize(["hello hadoop hello big data"])

word_counts = text \
    .flatMap(lambda line: line.split(" ")) \
    .map(lambda word: (word, 1)) \
    .reduceByKey(lambda a, b: a + b)

print(word_counts.collect())
# [('hello', 2), ('hadoop', 1), ('big', 1), ('data', 1)]

Step-by-step Explanation

StepRDD OperationPurpose
1parallelizeCreate RDD from input sentence.
2flatMapSplit line into words.
3mapConvert each word to (word, 1).
4reduceByKeyAdd counts for same word.
5collectTrigger execution and return result.

9. Lazy Evaluation, DAG, and Lineage

Interview Favorite

Spark does not execute transformations immediately. It records the transformations as a DAG, which means Directed Acyclic Graph. Execution starts only when an action is called.

textFile()
→
flatMap()
→
map()
→
reduceByKey()
→
collect()

Why Lazy Evaluation?

  • Spark can optimize the execution plan.
  • Spark can combine multiple transformations.
  • Spark avoids unnecessary computation.
  • Fault recovery is possible through lineage.
rdd = sc.parallelize([1, 2, 3, 4, 5])
filtered = rdd.filter(lambda x: x > 2)  # No execution yet
mapped = filtered.map(lambda x: x * 10) # No execution yet
print(mapped.collect())                 # Execution starts here
Interview line: Transformations are lazy because Spark builds a logical execution plan first. Actions trigger the job.

10. Partitions, Cache, Persist, Broadcast, and Accumulators

Performance

10.1 Partitions

Partitions are smaller chunks of an RDD. Spark runs tasks on partitions.

rdd = sc.parallelize(range(1, 101), 4)
print(rdd.getNumPartitions())  # 4

10.2 Repartition and Coalesce

OperationUseShuffle?
repartition(n)Increase or decrease partitions.Yes
coalesce(n)Usually decrease partitions efficiently.Usually avoids full shuffle
rdd2 = rdd.repartition(8)
rdd3 = rdd2.coalesce(2)

10.3 Cache and Persist

Use cache() or persist() when the same RDD is reused multiple times.

logs = sc.textFile("logs/app.log")
errors = logs.filter(lambda line: "ERROR" in line)
errors.cache()

print(errors.count())
print(errors.take(10))

10.4 Broadcast Variable

Broadcast sends a read-only value to all workers efficiently.

blocked_countries = sc.broadcast({"XX", "YY"})
transactions = sc.parallelize([("T1", "IN"), ("T2", "XX")])
valid = transactions.filter(lambda t: t[1] not in blocked_countries.value)
print(valid.collect())

10.5 Accumulator

Accumulator is used to aggregate counters across tasks.

invalid_count = sc.accumulator(0)
records = sc.parallelize([10, -5, 20, -1])

def validate(x):
    global invalid_count
    if x < 0:
        invalid_count.add(1)
    return x >= 0

valid_records = records.filter(validate).collect()
print(valid_records)
print("Invalid count:", invalid_count.value)

Count words and sort by count descending.

lines = sc.parallelize(["spark spark rdd", "python spark data"])

12. Interview Questions and Answers

15+ Q&A

1. What is RDD?

RDD is Spark’s immutable, distributed collection of elements processed in parallel. It supports fault tolerance using lineage.

2. Why is RDD immutable?

Immutability makes recovery easier. Instead of changing old data, Spark creates a new RDD from transformations.

3. What is lazy evaluation?

Spark delays execution of transformations until an action is called.

4. Transformation vs Action?

Transformation creates a new RDD. Action triggers execution and returns/writes result.

5. map vs flatMap?

map returns one output per input. flatMap can return multiple outputs per input and flattens them.

6. reduceByKey vs groupByKey?

reduceByKey aggregates locally before shuffle. groupByKey groups all values and may cause more network transfer.

7. What is lineage?

Lineage is the record of transformations used to create an RDD. Spark uses it to recompute lost partitions.

8. What is partition?

A partition is a chunk of an RDD. Spark creates tasks based on partitions.

9. Why avoid collect()?

collect() brings all records to the driver and can cause memory issues for large data.

10. What is cache?

cache() stores an RDD in memory for reuse, avoiding repeated recomputation.

11. What is persist?

persist() stores RDD using a selected storage level such as memory or disk.

12. What is shuffle?

Shuffle is data movement across partitions/nodes, usually needed for grouping, joining, and aggregation.

13. What is broadcast variable?

A read-only variable copied efficiently to worker nodes for lookup/reference data.

14. What is accumulator?

A shared write-only counter used by tasks to add values, commonly for metrics.

15. When should we use RDD over DataFrame?

Use RDD for low-level transformations, unstructured custom processing, or learning internals. Use DataFrame for structured data and optimization.

16. What is SparkContext?

SparkContext connects the driver program to the Spark cluster and is used to create RDDs.

17. What is a narrow transformation?

A transformation where each output partition depends on one input partition, such as map and filter.

18. What is a wide transformation?

A transformation where output partitions depend on multiple input partitions, usually causing shuffle, such as reduceByKey and join.
